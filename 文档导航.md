## 当前结论边界

v2 的角色是：

```text
无实测数据、面向真实仪器约束的模拟补强
```

它补充原来的工程逻辑和全量模拟结果，使相对模拟判断更可信；它不等同于真实采集、实测校准或实验验证。

允许的结论：

- 相对模拟证据；
- 受限仪器情景下的相对先验判断；
- 锁定路线在合成模型内的稳定性；
- 近壁网格、路线角色和窄通道风险先验的审计结论；
- 未来要进入实测或校准前必须补齐的 artifact 清单。

禁止的结论：

- 校准信噪比；
- 校准事件概率；
- 绝对检出限；
- 真实 EV 浓度；
- 生物特异性；
- 实测空白样本安全性；
- 路线直接升级；
- main-660 重新定义；
- selected-annulus 替代 all-crossing 主排序。

当前常用验证：

```bash
ruff check .
python -m pyright
python -m mypy
python tests/run_tests.py --workers 7
```

当前全量测试入口由 `tests/run_tests.py` 统一调度：非 AppTest 测试走 xdist 并行 lane，真实 Streamlit AppTest 保持 serial lane 并与并行 lane 重叠执行。`pyright` / `mypy` 目前检查配置中的 typed seed allowlist，用来防止新整理出的公共助手继续退化；全仓类型债尚未作为 release gate。`numba` 仍是可选加速依赖；缺失时会提示运行在无 JIT 加速路径，而不是静默降速。

源码整理路线图见 [docs/PROJECT_ORGANIZATION_ROADMAP_2026-05-09.md](./docs/PROJECT_ORGANIZATION_ROADMAP_2026-05-09.md)。当前 package-module 实现已经集中到 `nodi_simulator/`，根目录 package-module 兼容入口已退场；后续整理聚焦文档指针、archive-only 脚本和独立的 tools wrapper 迁移窗口。
