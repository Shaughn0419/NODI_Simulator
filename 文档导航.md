更新时间：2026-06-12。

本文是当前工程的读者入口与口径索引。若历史报告、旧 review bundle、旧计划文件与本文冲突，以本文列出的当前入口和 claim boundary 为准；历史文件只用于追溯 provenance，不再直接承载最终结论。

## 一句话结论

当前工程已经收口为一份 **无实测数据的 Level-1 相对/代理审计**：它可以回答“在当前 detector surrogate 与合成事件模型下，哪些候选路线更值得进入实测验证”，但不能授权任何 calibrated SNR、LOD、blank-FPR、真实浓度、EV 生物特异性、NODI 增强倍数或 detector-resolved 绝对胜者结论。

最终读法不是“单一波长/通道绝对冠军”，而是两个 candidate family：

- `404/W500`：fixed-view 下的高峰值候选，短波散射与较强参考场带来更高峰值，但仍受 detector surrogate、读出口径和噪声边界约束。
- `660/W800`：per-wavelength-view 下的稳健候选，参考场更温和、路线解释更保守，适合作为实测优先基线。

`488/532` 只保留为 control/trend 波长；`selected-annulus` 是 event-position analysis window，不是 BFP optical annulus。

## 当前结论边界

允许写：

- candidate families under detector surrogate；
- no-measured-data Level-1 relative/proxy evidence；
- 双口径并立：all-crossing 与 selected-annulus 都保留；
- `404/W500` 与 `660/W800` 的相对候选角色；
- T3/T4/Stage-1 在当前 artifact 上复现出的审计结果；
- R1 与 C/D×V2 是收窄封板门之外的 deferred cells。

禁止写：

- detector-resolved 绝对 winner；
- 绝对跨波长 winner；
- calibrated SNR、LOD、blank-FPR、真实浓度、真实检出概率；
- biological specificity；
- NODI 外差增强倍数；
- selected-annulus = BFP optical annulus；
- R1 不变性已验证；
- C/D×V2 已全覆盖；
- 488/532 已成为推荐波长。

## 历史报告使用规则

历史报告保留是为了 provenance，而不是为了让旧结论继续并列竞争。读到以下旧口径时，应按当前状态降级：

- “660 conservative main route / 404 sidecar”：已过时。当前是 `404/W500` 与 `660/W800` 两个 candidate family，取决于 fixed/per-wavelength view。
- “D1400/D1500 是最终推荐深度”：已过时。当前深度建议是 D900–D1200 保守工程带；D1400/D1500 是更高峰值或历史 per-view 背景，不是 mandatory final design。
- “selected-annulus 主排序可替代 all-crossing”：错误。两者是不同事件窗口口径，必须并立。
- “补齐 A/B/C/D × R1/R2 × V1/V2 后才封板”：已由 2026-06-12 owner 裁决收窄。R1 与 C/D×V2 是 deferred cells，不阻塞 no-data 定稿。

## 测试与操作

常用验证：

```bash
ruff check .
python -m pyright
python -m mypy
python tests/run_tests.py --workers 7
```

当前全量测试入口由 `tests/run_tests.py` 统一调度：非 AppTest 测试走 xdist 并行 lane，真实 Streamlit AppTest 保持 serial lane。`numba` 是可选加速依赖；缺失时运行无 JIT 路径。挂载盘上可能出现的 AppleDouble `._*` 元数据文件不属于源码或文档清单。
