## 0. 先给结论

先把证据等级说清楚：报告现在有两层证据。**Stage B6** 是 per-wavelength gold-normalized 的 Tsuyama 诊断 / 几何排序口径；**Stage B7** 是固定 660 金颗粒基准的跨波长修正口径。B7 已经完成，因此 404 vs 660 的低事件数合成读法不再停留在“等待重算”的状态。

- **B7 固定 660 基准后，推荐合格集（404/660）顶部转向 404 nm / 500 nm 宽度族。**
- 默认 B7 中心可以写成 **404 / 500 x 550 nm**；小 EV 与尖锐小 EV 压力读法下，**404 / 500 x 900 nm** 是并列核心候选。
- **660 / 800 x 1000-1400 nm** 仍保留为 B6 gold-normalized / Tsuyama-lineage 对照家族，以及长波保守候选；但它不再是 B7 固定 660 基准下的首选。
- **404 nm 现在不是“保留等待 B7”的旁路，而是 B7 低事件数修正口径下的主候选。**这个结论仍是单 seed、1000 events/case 的合成设计证据，不是实测校准，也不是 10000 events/case 终验证。
- **488 / 532 不进入最终推荐**：它们只能作为趋势 / 对照。即使 488 在某些表中很高，也不能越过波长治理规则。
- **金颗粒行不参与 EV 推荐**：金颗粒只用来检查 Tsuyama 锚点和参数来源链。

当前证据仍是 **每工况 1000 个事件、单随机种子、低事件数设计证据**。它不是实测校准，也不是每工况 10000 个事件的最终验证；但在“固定 660 基准修正”这个问题上，B7 已经给出方向性更新。

## 1. 读者真正关心的问题：一条路线为什么能检测？

一条通道路线的检测率不是一个黑箱数字。它大致沿着下面的链条形成：

1. **粒子本征散射（Mie / 散射阶段）**：粒子在某个波长下本来能散射多少光。
2. **参考场幅值**：通道几何和光学模式给出多强的参考光。
3. **干涉峰值**：散射场与参考场叠加后，事件峰值有多高。
4. **通过时间**：粒子在光斑 / 通道读出区域停留多久；tau=1 ms 时，通过时间决定有效读出窗口。
5. **噪声与阈值**：峰值要穿过检测阈值；平均噪声余量代理高不等于每个事件都过阈值。
6. **选定环带分母**：口径 B 只把 Tsuyama 语义下的选定探测环带作为主读法。
7. **治理过滤**：404/660 才能进入最终推荐；488/532 只能作对照；金颗粒不能混入 EV 推荐。

### 1.2 峰值增强是不是简单相乘？

不是。一个直觉式写法是：

```text
观测峰值 ≈ |E_ref + E_sca|^2 - |E_ref|^2
        ≈ 2 * A_ref * E_sca * cos(phase) + |E_sca|^2
```

所以本征散射和参考场会共同推高峰值，但最终峰值还会受到相位、位置、通道模式、选定环带、通过时间和锁相读出的影响。**不能把散射增强倍数、参考场增强倍数、峰值增强倍数简单相乘后当检测率。**

### 1.3 为什么峰值高，噪声仍然重要？

有三个原因：

- 检测是事件级阈值判断，不是只看平均峰值。小 EV、边缘轨迹、相位不利事件会拉低检测率。
- 全穿越分母包含很多不在选定环带的事件；它们的参考场 / 峰值条件不如主环带。
- 当前噪声和参考场仍是合成模型，没有接入实测空白噪声、后焦面 / 感兴趣区域图像、锁相 / 记录器轨迹。因此报告只能声明“合成相对设计证据”。

### 3.0 先解释一个容易误读的现象：为什么 404 和 660 差异不大？

第 3 章现在同时列出两种量：**报告 88 式 strict physics 阶段量**和 **B6 归一化运行量**。这里的 `Csca` 按报告 88 的 reader 口径书写；散射强度代理来自 `E_sca_normalized^2`，而 `E_sca_normalized` 是相对基准金颗粒归一化后的检测方向散射场。

| 404 nm | 7.10x | 2.66x | 短波本征散射最强 | 每波长 Au 归一化、参考场、通过时间、相位 / 轨迹、选定环带 |
| 488 nm | 3.34x | 1.83x | 中间短波对照 | 同上；治理上仍不进入最终推荐 |

这就是为什么两者看起来不一样：当前运行配置使用每波长归一化，每个波长都会用同一基准金颗粒重新计算 `E_sca_ref`。这样做适合 Tsuyama anchor 的相对读法，但会压平一部分绝对波长散射差异。换句话说，**404 的绝对 Csca 可以明显大于 660，但归一化后进入检测链条的散射代理不一定大很多。**

从批判性审查看，这个归一化**不是无条件正确**。它的合理用途是“把各波长都放到各自的金颗粒锚点尺度上，做 Tsuyama lineage 的相对比较”；它的风险是“把 EV 自己的短波散射优势除掉一部分”。因此本报告不能只看归一化散射代理来判断短波是否有物理优势，必须同时列出 `Mie Csca`，并把当前 B6 结论写成 gold-normalized 设计证据，而不是跨波长绝对最终证据。

复核结论分两层：**没有发现报告聚合错误，但发现当前归一化口径不足以说服读者接受跨波长最终结论。**平均峰值差异偏小主要来自四件事：每波长基准归一化压平绝对 Csca；峰值均值只统计检测到峰的事件；相干干涉项包含相位和探测算子，不是 `Csca x A_ref`；404 通过时间更短，检测率被拉回。

所以这组表应这样读：

- **404 的绝对 Csca 确实更大**：按报告 88 的 reader 阶段量口径，404 / 660 写成 `7.10x`；当前 40-300 nm biomimetic EV 全粒径 panel 的 Csca 均值相除约为 `1.53x`，只作为聚合审查脚注，不放在主表 Csca 列里。
- **但 404 的 B6 归一化散射代理不一定更大**：在 800 nm 宽度族里，404 的 B6 归一化散射强度代理约为 660 的 0.93x。这是 Au anchor 归一化后的运行量，不是本征散射阶段量。
- **在当前 per-wavelength gold-normalized 口径里，404 的峰值优势主要来自参考场，而不是归一化散射代理本身**：例如 800 x 1100 下，404 的参考场是 660 的 1.29x，但平均峰值只到 1.10x。
- **峰值不是简单相乘**：平均峰值来自相干干涉项、相位、位置和探测算子，不能用“散射代理 x 参考场”直接推出。
- **检测率还被通过时间拉回**：404 的通过时间通常只有 660 的约 0.60-0.62 倍；tau=1 ms 下可积分的有效时间更短。
- 第 3 章后续表格同时保留 `Csca（报告88阶段量）` 和 `B6归一化散射强度代理`，目的就是把“本征散射阶段量”和“检测链条里的归一化运行量”分开。
- **结论已经按 B7 修改**：当前 B6 可以作为 Tsuyama/gold-normalized 诊断和几何排序证据；404 vs 660 的固定 660 基准修正读法以 B7 为准。

- 大 EV 的局部噪声余量代理很高，但检测率仍不会自动等于 100%，因为全穿越分母包含不利位置和不利相位事件。
- 选定环带检测率通常高于全穿越检测率，因为它只看 Tsuyama 语义下更有利的探测环带。

## 8. Tsuyama 论文怎样约束本报告？

Tsuyama 论文不是“把结论直接搬过来”的校准证书。它们给口径 B 提供四类约束：

| 约束 | 对本报告的作用 | 不能外推成 |
| --- | --- | --- |
| 660 nm NODI 读出背景 | 支持 660 作为论文中心波长 | 660 在所有真实实验中必然最佳 |
| 选定探测环带 | 支持选定环带主指标 | 实测事件概率 |
| 1-2 ms 时间常数背景 | 支持当前 tau=1 ms 运行纪律 | 旧 2 ms 可改名成 1 ms |
| Au/Ag 锚点 | 支持 B1 参数来源链诊断 | 金颗粒行可混入 EV 推荐 |
| POD / thermal 论文 | 提供边界和对照 | POD 检出限 可转移到 EV NODI |
| 每波长金颗粒锚定 | 支持论文 lineage 和锚点诊断 | 仅凭 B6 就说明 EV 跨波长推荐已经完成 |

当前最强、但仍然保守的说法应分 B6 / B7 两层：

> 在 tau=1 ms、单随机种子、每工况 1000 个事件的合成 EV 全网格下，Stage B6 的 per-wavelength gold-normalized 口径支持 660 nm / 800 nm 宽度族和 1000-1400 nm 高度带作为 Tsuyama-lineage 对照；Stage B7 的固定 660 金颗粒基准修正口径则把推荐合格集顶部改为 404 nm / 500 nm 宽度族，中心候选为 404 / 500 x 550 与 404 / 500 x 900。488/532 nm 仍只作趋势 / 对照，金颗粒仍只作锚点诊断。该结论不是实测校准，也不是每工况 10000 个事件的最终验证。

## 9. 发布边界

### 9.1 可以说

- Stage B6 tau=1 ms EV + 金颗粒全网格已完成 32,032 行。
- Stage B7 固定 660 金颗粒基准修正重算已完成 32,032 行。
- 当前 EV 推荐只使用 EV（exosome 标记）行。
- 当前低事件数 B6 gold-normalized 设计证据支持 660 / 800 x 1000-1400 家族作为 Tsuyama-lineage 对照。
- 当前低事件数 B7 fixed-660 设计证据支持 404 / 500 x 550-900 家族作为口径 B 主候选。
- 404 的绝对 Mie 散射优势在 B7 中转化为推荐合格集顶部优势；这仍需 10000 events/case、更多 seed 和实测噪声 / 参考场校准复核。
- 488/532 是趋势 / 对照，不是最终推荐。
- 金颗粒只做 Tsuyama 锚点诊断。

### 9.2 不能说

- 不能说这是每工况 10000 个事件的最终验证。
- 不能说这是实测校准。
- 不能说 `660 / 800 x 1100` 是 B7 后的物理绝对最佳。
- 不能说当前 B6 已经证明 660 在绝对 EV 跨波长灵敏度上最终胜过 404；B7 的低事件数修正读法恰好不支持这个说法。
- 不能说 404 已经通过实测校准最终替代 660；当前只是固定 660 基准、低事件数合成证据下的主候选。
- 不能把 488/532 写成最终推荐波长。
- 不能把金颗粒行混入 EV 行选胜出项。
- 不能把 B1 锚点几何直接写成 B2 EV 推荐。
- 不能把旧 2 ms 全网格改名成当前 1 ms 证据。
- 不能把每波长金颗粒归一化后的散射代理当作 EV 绝对散射强度。

| 目标 | 下一步 | 解决的问题 |

### 10.1 B7 结果现在怎样使用

B7 已经不是待办项，而是当前跨波长修正口径。使用时要守住下面几条边界：

1. **B7 是 fixed-660 修正口径，不是实测单位口径**：它修正了 per-wavelength Au normalization 的跨波长问题，但还没有引入真实探测器响应、真实空白噪声或真实后焦面图像。
2. **B6 不删除**：B6 名称应写成“per-wavelength gold-normalized Tsuyama diagnostic lane”，用于解释锚点诊断和同波长几何排序。
3. **B7 当前首选应写成族而不是孤点**：404 / 500 x 550 在 uniform 与 broad EV 假设下最高；404 / 500 x 900 在 small EV 与 sharp MSC-sEV 假设下最高。
4. **660 保留为保守长波对照**：660 / 800 x 1000-1400 仍是 B6 稳定家族，也适合作为后续实测对照路线。
5. **下一次签字条件**：只有当 B7 的 10000 events/case、seed 稳健性、实测噪声 / 参考场校准方向一致时，才能把“404 主候选”升级为最终实验路线。
