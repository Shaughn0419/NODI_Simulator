# 计算侧收口总结与工程瘦身记录

> Codex 执行修订：2026-05-06。本文是 Phase 2 / Phase 2.11 后的计算侧收口说明，也是本轮工程瘦身记录。它不替代 `reports/48_*` 和 `reports/49_*` 的科学分析，只给后续接手者说明：哪些计算结论已经够用、哪些探索停止、哪些临时文件已删除。

## 1. 计算侧最终结论

当前可以停止 Tsuyama 2022 selected-annulus paper-fit / estimated-parameter search。理由不是模型已经绝对复现论文，而是低自由度、可解释的计算方向已经走到边界：

- formula-consistent Table S1 Ag/Au signal 已基本对齐；strict Table S1 column-ratio 保留为 diagnostic lens。
- detection 在 asymmetric Au20 口径下不是主 blocker：Au20 lower miss 是 weak-SNR warning，Au20 over-detection 仍是 hard guard。
- raw Au size-response 仍是主 residual，尤其 `40-60 nm` pair 过陡；D2.1 best raw exponent 仍约 `3.05`。
- single global response-compression `gamma=0.749` 后 D2.1 best score 为 `2.033`，接近但未达到 bounded partial。
- maximal upper-bound 可以更接近论文数值，但需要 higher-DOF empirical terms，不能签 raw calibration。

因此最终 release 仍是：

```text
negative_or_diagnostic_result_only
```

它不回写 EV full-grid，不修改 selected-annulus `0.5-0.8`，不改变 global material defaults，也不阻塞 660/404 第一轮实验准备。

## 2. Stop-Decision 后新增的两个边界检查

### 2.1 ET-2030 / LI5640 Instrument Feasibility

新增工具：

```text
tools/instrument_hardware_feasibility.py
```

输出：

```text
results/instrument_hardware_feasibility_v1/
```

结论：

- current-input / low-noise TIA 路线在 estimated prior 下全部为 `comfortable_margin`。
- 50 Ω voltage path 有 `211/216` rows 为 `below_minimum_sensitivity`，`5/216` rows 为 `near_minimum_sensitivity`。
- 这支持实验优先核对 current-input / TIA 接法、量程和 lock-in 设置。
- 该检查只是 instrument feasibility estimate，不解锁 absolute SNR / detector-unit calibration。

### 2.2 Paper-Statistics / Size-Distribution Sensitivity

新增工具：

```text
tools/tsuyama_paper_statistics_sensitivity.py
```

该工具要求显式传入 Phase 2.10 decomposition CSV，避免后续重跑时静默复用旧 D2.1 输入。

输出：

```text
results/tsuyama_paper_statistics_sensitivity_v1/
```

结论：

- 用 Phase 2.10 decomposition 估算有限计数、IQR trimming、vendor size distribution 需要贡献多少 flattening。
- `274/288` rows 为 `paper_statistics_unlikely_alone`，`14/288` rows 为 `paper_statistics_borderline`。
- 最佳 D2.1 local-SNR 仍需要中位约 `30.6%` high-size signal suppression；peak-height 需要约 `31.8%`。
- 这些统计效应可能贡献一部分，但没有 event-level pulse 或 measured size distribution 时，不能单独解释 raw Au size-response mismatch。

## 6. 后续建议

计算侧不再建议继续：

- broad D2 refphase / BFP / collection sweep
- noise / threshold / blank surrogate sweep
- selected-annulus window sweep
- per-diameter / per-geometry correction
- detection logistic remap
- 调 score 权重让 `2.033` 刚好过线

后续如果继续投入，优先转向实验和硬件链：

- ET-2030 -> LI5640 current input / TIA 接法与量程确认
- blank trace by wavelength / geometry
- BFP / slit / ROI image
- Au20/30/40/60 raw pulse traces
- LI5640 filter / sampling / demod metadata
- TEM / DLS / vendor batch size distribution

这些数据进入后，再重启 detector-unit / instrument-aware calibration；在此之前，Tsuyama numeric fitting 搜索保持停止。
