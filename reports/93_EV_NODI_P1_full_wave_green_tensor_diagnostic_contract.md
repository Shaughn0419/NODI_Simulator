# EV/NODI P1 Full-Wave Green-Tensor Diagnostic Contract

Date: 2026-05-10

Status:

```text
no-solver rank diagnostic; no solver execution
```

This document narrows the first P1 lane from the physical-ceiling extension
registry into a full-wave / Green-tensor diagnostic contract. It is still a
surrogate-risk reduction artifact. It does not change the P0 release
conclusion.

## Boundary

The lane may later compare rank evidence from a bounded Green-tensor diagnostic
against the P0 scalar and BFP rank evidence. It may not convert the review
package into calibration.

Required declarations:

```text
calibrated_claim_allowed = false
p0_release_conclusion_changed = false
physical_ceiling_role = surrogate_risk_reduction_only
```

SNR remains blocked. LOD remains blocked. EV concentration remains blocked.
Biological specificity remains blocked. Detector-voltage prediction remains
blocked.

## Inputs

Required P0 sources:

| source | role |
|---|---|
| `results/post_v2_mandatory_audit/top_candidate_mandatory_audit.csv` | route keys, comparison strata, existing rank evidence, and claim blocker columns |
| `results/post_v2_mandatory_audit/top_candidate_pairwise_rank_inversion.csv` | pairwise inversion evidence |
| `results/post_v2_mandatory_audit/bfp_roi_operator_summary.csv` | audited BFP Jacobian provenance |

Future solver config, mesh manifests, or operator exports require separate
authorization. They are not generated by this contract stage.

Forbidden inputs:

```text
measured blank traces
standard-particle calibration curves
detector-gain calibration
route-specific manual multipliers
raw arbitrary-unit magnitudes as final gates
```


## Gate Policy

Allowed future gates:

```text
rank
rank_percentile
pairwise_inversion
```

Raw arbitrary-unit magnitudes may appear only as diagnostic traces and may not
be final gates. Small-stratum changes are secondary notes; the primary
comparison stratum remains `all_ranked_routes`.

## Jacobian Layering

The v1 source library remains:

```text
v1_bfp_to_angle_jacobian_applied = false
```

The post-v2 audit sidecar remains:

```text
audit_bfp_jacobian_applied = true
```

The full-wave contract may reference audit-sidecar Jacobian provenance, but it
must not rewrite the v1 source-library Jacobian flag.

## Stop Rule

Stop before implementation if any of the following appears:

```text
solver execution
new Green-tensor mesh generation
measured-data ingest
route promotion
main-660 redefinition
raw magnitude used as a final gate
v1 event-count expansion
```

The next allowed step is a stub validator or generator that emits only an empty
schema manifest. It is not a full-wave implementation.
