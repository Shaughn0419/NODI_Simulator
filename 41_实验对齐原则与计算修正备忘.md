## 1. Tsuyama 六篇论文给出的当前硬约束

先说当前最重要的分层：

- 对 current NODI/exosome 主线最直接的硬约束，仍主要来自 `2020 diffraction / 2022 NODI / 2024 POD+NODI`
- `2019 POD / 2020 counting POD / 2020 solvent-enhanced POD` 这三篇没有再额外引出新的 current mainline must-fix，但它们补强了 POD 支线的适用范围、热扩散语义、以及“不能把 POD detectability 直接外推到 NODI detectability”这条边界

### 1.1 2020：检测必须落在衍射区，而不是直透射区

Tsuyama 2020 已经把 NODI/POD 的基本几何说清楚了：

- 收集端是 `NA = 0.90`
- transmitted beam 需要被 slit 切掉
- 有用信号来自 nanochannel diffraction region

这意味着 current collection 逻辑必须至少尊重两个事实：

- 不同 `W / λ` 会改变衍射角和 slit 最优区域
- 如果一阶衍射本身收不进物镜，参考场就不能继续保留非零

### 1.2 2022：照明端是 `20x, NA = 0.45`，spot size 约 `2 μm`

Tsuyama 2022 给出的实验条件是：

- illumination objective: `20x, NA = 0.45`
- calculated spot size: about `2 μm`
- collection objective: `NA = 0.9`
- slit width: `1 mm`
- pinhole: `400 μm`
- nanochannel: `800 / 1200 nm` wide, `550 nm` deep
- measured / compared wavelengths include `488 / 532 / 660 nm`; the dual-wavelength classification branch is `488 / 532 nm`

所以对当前工程最关键的不是“照明是不是高斯”，而是：

- `x/z` 横截面上它对 `700–800 nm` 量级通道近似 overfill
- `y` 方向穿过焦区的 transit window 仍然有限

### 1.3 2024：输运是 pressure-driven，典型流速约 `0.2 mm/s`

Tsuyama 2024 又补充了当前输运判断最关键的三点：

- optical split: `532 nm` excitation for POD, `660 nm` probe for POD/NODI signal measurement
- nanochannel: `800–1200 nm` wide, `550 nm` deep
- 流体驱动方式是 pressure-based
- `100 kPa` 时典型流速约 `0.2 mm/s`
- individual nanoparticles pass through the detection region within about `10 ms`
- 在这个时间尺度上，diffusion length can be comparable to or greater than channel size

注意：`800x710 nm` 是 2020 POD counting paper 的通道几何，不是 2022/2024 NODI 的原文几何；工程中只能作为 POD geometry bridge / sensitivity case，不能再标记为 NODI exact geometry。

这几句话决定了我们不能同时犯两种错：

- 不能把输运永远当成纯 `plug flow + no diffusion`
- 也不能在当前轨迹结构里随手塞一个“flux-biased initial position”就当成严格修正

## 2. 已经写入当前代码的对齐项

### 2.1 `NA cutoff`：已落实

当前 `nodi_simulator/reference_field.py` 已经在参考场链路里执行 collection `NA` 截止。

核心红线仍然是：

`W_min = λ / NA_collection`

对 `NA_collection = 0.9`：

- `404 nm` -> `W_min ≈ 449 nm`
- `488 nm` -> `W_min ≈ 542 nm`
- `532 nm` -> `W_min ≈ 591 nm`
- `660 nm` -> `W_min ≈ 733 nm`

因此：

- `660 nm + 700 nm width` 必须被截断
- `660 nm + 800 nm width` 才进入可收集区

这条现在已经是代码硬约束，不再只是文档提醒。

### 2.2 illumination / detection 几何已解耦

当前 illumination geometry 不再和 detection / shared beam waist 混用。

现在的口径是：

- illumination 端优先读取独立 illumination waist
- 若未显式给出，就按 `0.61 * λ / NA_illumination` 生成 decoupled illumination surrogate
- `phi_beam`、focus-crossing surrogate 和 overfill 判断都走这套 illumination geometry

这一步已经把“照明 `NA=0.45`”和“收集 `NA=0.9`”这两套物理角色拆开了。

### 2.3 overfill 现在只抹平 `x/z`，保留 `y` 向 transit window

当前 `illumination_mode=”overfill”` 的真实语义是：

- `x/z` 上近似均匀照明
- `y` 向仍保留穿过焦区的有限包络

这一步修掉了早期那个”把整个包络都压成 1，导致 transit time 被误读成整段 trace 时长”的漏洞。现在代表 case 的 transit time 会回到 Tsuyama 论文给出的 `~10 ms` 量级，而不是错误的 `200 ms`。

### 2.4 Gouy 相位去重：已落实（2026-04-12）

`nodi_simulator/illumination.py` 输出 `phi_beam_gouy = arctan((y-y_f)/z_R)` 作为照明侧穿焦相位。  
`nodi_simulator/scattering_trace.py` 中 `_surrogate_focus_crossing_phase()` 也返回 `arctan((y-y_f)/z_R)` 相同公式。

在 `relative_surrogate` 模式下，旧代码把两者同时叠加入 `phi_sca_unwrapped`，等价于 2× arctan，Gouy 相位幅度虚增 2×。

**修正（`_gouy_dedup_active = True` 当 `phase_model == “relative_surrogate”`）：**

- 信号路径仅使用 `phi_beam_curv + phi_focus_crossing`（曲率 + 一次 arctan）
- 不再同时加入 `phi_beam_gouy`
- 诊断字段 `phi_beam_gouy_rad` 仍然输出，用于照明侧审计
- `phi_gouy_ref_rad / phi_gouy_sca_rad / delta_phi_gouy_rad` 现在按
  **实际进入 `E_ref / E_sca` 的 active 相位贡献**导出：
  - `phi_gouy_ref_rad = 0`（reference trace 未携带 `phi_beam_gouy`）
  - `phi_gouy_sca_rad = phi_focus_crossing`（`relative_surrogate`）
  - `delta_phi_gouy_rad = phi_gouy_sca_rad - phi_gouy_ref_rad`

数值验证（λ=660 nm, NA_ill=0.45, n=1.33）：

| 指标 | 修正前 | 修正后 |
|------|--------|--------|
| FWHM 内相位摆幅 | 33.4° | 16.7° |
| 全轨迹相位摆幅 | 165.7° = 0.92π | 82.9° = 0.46π |
| 峰值 NODI 信号（外泌体）| 0.33 | 0.18 |
| 拓扑零交叉次数 | 1 次 | 1 次（不变）|

`constant` / `axial_path` 模式不受影响（`phi_focus_crossing = 0`，`_gouy_dedup_active = False`）。

## 3. 当前输运口径的最终判断

### 3.1 现行全量主线不是 `plug`

这里需要把两个默认入口分开说清楚：

- `nodi_simulator/data_objects.py::DEFAULT_SIM_CFG`
  仍保留更保守的最小审计入口，默认 `flow_profile_model="plug"`
- `dashboard/config.py::DEFAULT_SIM_CFG`
  才是当前 dashboard / precompute / 单案例页的现行主线，默认已经是：
  - `flow_profile_model="rect_series"`
  - `include_diffusion=True`
  - `diffusion_hindrance_model="anisotropic_tensor_surrogate"`

所以“当前标准全量计算是否仍在用 plug flow”这个问题，答案是：不是。

### 3.2 为什么这轮没有再加一层独立的 flux-weight correction

这次重新核对 Tsuyama 2024 和相关颗粒输运文献后，我的结论是：

**在当前轨迹实现里，不能把‘flux weighting’简单写成一个新的初始位置偏置。**

原因有三个：

1. 当前轨迹在到达焦区前还有约半段 trace 的预跑，粒子不是从“检测平面入口”直接开始模拟。  
2. Tsuyama 2024 明确指出，粒子在约 `10 ms` 内的 diffusion length 已可与通道尺度同量级；而当前轨迹在进入焦区前的预跑时间更长，`x/z` 会进一步重混。  
3. 对 pressure-driven flow，严格的 flux weighting 应该作用在“穿过观测平面的事件通量”上，而不是直接等同于“模拟起点的初始横截面采样”。  

因此，如果以后要继续升级输运模型，正确方向应该是：

- 以 plane-crossing / entrance-conditioned transport 为中心重写事件采样
- 和 diffusion、hindrance、focus-crossing 时间定义一起联动设计

而不是只额外加一个 `flux_weighted_initial_position` 之类的局部补丁。

我的当前判断是：

**2026-04-12 补充**：Gouy 相位双重叠加（§2.4）是本轮额外确认并修正的 bug，修正后全量重算条件仍然成立。

## 5. 当前仍然保留的模型边界

以下内容仍然值得在后续版本继续追，但它们不是本轮全量重算前的阻塞项：

1. `channel_diffraction + pupil_slit_surrogate` 仍不是 full-wave slit-position solver  
2. `channel_angular_surrogate` 仍不是严格 blank-channel field solver；若拿到更完整的 blank 标定，优先走 `calibrated_lookup`  
3. 若以后要认真升级 pressure-driven event statistics，应从 crossing-conditioned transport 入手，而不是额外叠加一个简单的初始位置偏置  
4. Tsuyama gold validation 的 `paper_aligned` 判据修正当前只作用在验证支线；主线全量重算仍按 current mainline 口径执行，不应把这条 scoped fix 直接偷渡成全局默认语义
