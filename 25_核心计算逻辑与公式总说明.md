# 核心计算逻辑与公式总说明

本文只说明当前代码实际采用的工程计算逻辑。它不记录改造过程，也不展开所有辅助诊断模块清单；重点是把“一个 case 怎么算、一个 sweep 怎么跑、结果库怎么保存和读取、哪些结论能解释到什么程度”讲清楚。

当前主链可以压成一句话：

`粒子本征散射 -> 探测算子收集 -> 通道参考场干涉 -> 噪声与读出 -> 阈值和脉冲提取 -> batch 统计 -> 工程 gate 与排序 -> 保存为可审计结果库`

输出不是单一 `Csca`，而是一组分层结果：

- 物理层：`Csca / Cext / Cabs / S1 / S2 / E_sca_detected / E_ref`
- 事件层：`signal_trace / signal_detect / threshold / pulse features`
- batch 层：`detection_rate / stable_detection_rate / phase_flip_fraction / mean_peak_margin_z / AUC / d'`
- 排序层：`score / engineering_score / final_engineering_score`
- 治理层：`output_claim_level / minimum_output_schema_status / final_green_eligible / primary_blocker_summary`
- 结果库层：`summary.csv / design_postprocess.csv / compact.pkl / meta.json / result_health.json / runtime_performance.json`

---

## 1. 运行入口

### 1.1 单 case / 交互路径

单 case 路径用于 inspector、single-case calculator 和临时机制复核。它的特点是：

- 可以保留完整 event trace 和细节诊断。
- 适合解释“这个 case 为什么这样”。
- 默认不启用 `event_block_v3`，full diagnostics 需要逐事件明细。

### 1.2 sweep 路径

核心入口是：

`parameter_sweep.run_parameter_sweep(...)`

它枚举：

`particle_types × wavelength_list_m × width_list_m × depth_list_m`

每个 case 会经过完整单 case 计算，然后进入评分和排序。当前 sweep 的重要约束是：

- `theta_grid_rad` 必须显式传入，不能为 `None`。
- 默认 `allow_partial=False`；任一 case 失败会让整个 sweep 失败，不再静默保存偏置子集。
- 只有显式 `allow_partial=True` 时才允许保存成功子集，并且 completion policy 会记录 partial 状态。
- parallel worker 会限制 BLAS/OpenMP 线程，避免 worker 进程再叠加多线程抢占。

### 1.3 precompute / 结果库路径

正式全量重算入口是：

```bash
PYTHONPATH=. python -m dashboard.precompute \
  --grid ev_design \
  --particle-profile full_range_biomimetic_exosome_with_anchors \
  --tag full_range_biomimetic_exosome_with_anchors_10000e \
  --workers 8 \
  --freeze-probe-report \
  --artifact-profile standard \
  --progress-interval 2 \
  --resume \
  --checkpoint \
  --checkpoint-batch-size 100 \
  --checkpoint-flush-interval 5 \
  --output results
```

这条路径在 sweep 之上增加：

- safe tag / prefix 校验
- progress JSON
- checkpoint chunk
- crash-safe resume，并校验 checkpoint manifest 与当前 `grid / config_tag / particle_profile / total_cases` 一致
- artifact profile
- schema `1.24` metadata
- result health report
- runtime performance report
- 可选 freeze probe
- 成功后的 transient checkpoint/progress 清理

正式结果库应优先从这条路径生成，不应手写循环拼 CSV。

## 3. 输入对象

核心对象来自 `nodi_simulator/data_objects.py`：

- `Particle`
  粒子半径、材料、材料库开关、core-shell 参数。
- `Medium`
  介质折射率、粘度、温度。
- `Channel`
  通道宽度 `W` 和深度 `H`。
- `OpticalSystem`
  波长、beam waist、焦点位置、收集角、收集 NA。
- `SimulationConfig`
  reference、输运、噪声、读出、阈值、事件数、评分和 gate 配置。

一次 case 的最小身份是：

`particle_name + wavelength_m + width_m + depth_m + simulation_config`

precompute 会把这个身份映射成稳定 `case_key`，用于 checkpoint 去重、resume 和随机数策略。

## 4. 单 Case 计算链

### 4.1 材料与 Mie 本征散射

粒子和介质进入 Mie 层的量是：

`ñ_p(λ) = n_p(λ) + i k_p(λ)`

`n_m(λ) = Re[ñ_m(λ)]`

对均匀球：

`k_m = 2π n_m / λ0`

`x = k_m a`

`m_rel = ñ_p / n_m`

`nodi_simulator/mie_engine.py` 计算 Mie 系数 `a_n, b_n`，得到：

`Qext = (2/x^2) Σ (2n+1) Re(a_n + b_n)`

`Qsca = (2/x^2) Σ (2n+1)(|a_n|^2 + |b_n|^2)`

`Cgeo = πa^2`

`Csca = Qsca · Cgeo`

`Cext = Qext · Cgeo`

`Cabs = Cext - Csca`

角散射振幅保留复数：

`S1(θ), S2(θ)`

非偏振平均的角散射截面是：

`dCsca/dΩ = (|S1|^2 + |S2|^2) / (2 k_m^2)`

场幅值代理：

`Esca_unit_amp(θ) = sqrt(dCsca/dΩ)`

若粒子是 core-shell，则走 `mie_core_shell_coefficients(...)`，但输出仍统一成 `Csca / Cext / Cabs / S1_complex / S2_complex / Esca_unit_amp`。

### 4.2 探测算子与 baseline 归一化

系统不是直接使用完整角散射图，而是用共享探测算子压缩成检测场：

`E_sca_detected_complex = L_det[field_sca(θ, φ)]`

当前默认：

- `collection_angle_model="channel_diffraction"`
- `collection_integration_mode="pupil_slit_surrogate"`
- `scattering_projection_mode="parallel"`

`channel_diffraction` 的中心角来自：

`λ_eff = λ0 / n_m`

`theta_base = arcsin(m · λ_eff / W)`

`theta_center = center_scale(W,H) · theta_base`

`pupil_slit_surrogate` 用有限 `theta / phi` 接受窗和 throughput 近似物镜、狭缝、pinhole 的收集趋势。它不是完整 pupil integral。

散射投影默认取平行通道：

`field_sca(θ) = S2(θ) / k_m`

然后用 baseline particle / baseline channel / 同一个探测算子做归一化：

`E_sca_unit_normalized = E_sca_detected_complex / E_sca_ref(λ)`

当前默认 `normalization_mode="per_wavelength"`，所以每个波长有自己的 baseline。

### 4.3 Reference 场

NODI 主链看的是 reference 和 scattering 的干涉，不是单独的 `|E_sca|^2`：

`signal_trace = |E_ref + E_sca|^2 - |E_ref|^2`

当前 reference model 支持：

- `constant`
- `geometry_scaled`
- `channel_angular_surrogate`
- `paper_aligned_phase_filter`
- `calibrated_lookup`

当前主线默认是：

`reference_model="channel_angular_surrogate"`

这一路用通道角谱 surrogate 生成参考场，再用同一个探测算子 collapse：

`E_ref ~ L_det[E_diff,ch]`

reference 还会解析更高层 route：

- `calibrated_primary`
- `paper_aligned_comparison`
- `engineering_fallback`
- `legacy_debug`

route 决定 claim level 和 NA policy。`engineering_fallback / legacy_debug` 下保留 hard NA guardrail；`paper_aligned_comparison / calibrated_primary` 下记录 cutoff diagnostics，而不是直接把有限 NA 解释成 hard-zero 真值。

case 级 reference 可推进为事件级：

`E_ref_trace_complex(t)`

当前默认：

`reference_spatial_mode="cross_section_surrogate"`

它让 reference 幅值和相位随横截面位置 `(x(t), z(t))` 有最小起伏。

### 4.4 粒子输运

坐标约定：

- `x`：通道宽度方向
- `y`：流动方向
- `z`：通道深度方向

粒子中心可达范围：

`x ∈ [-(W/2-a), +(W/2-a)]`

`z ∈ [-(H/2-a), +(H/2-a)]`

初始位置采样支持：

- `uniform`
- `uniform_accessible_area`
- `center_biased_surrogate`
- `flux_weighted`

precompute 默认用：

`event_sampling_policy="sobol_stratified"`

轨迹流型支持：

- `plug`
- `parabolic_rect`
- `rect_series`

dashboard/precompute 默认是：

`flow_profile_model="rect_series"`

若开启扩散，会在 `x/z` 上加入 Brownian 随机游走，并按 `diffusion_hindrance_model` 处理近壁抑制。precompute 默认：

`include_diffusion=True`

`diffusion_hindrance_model="anisotropic_tensor_surrogate"`

### 4.5 照明场

`nodi_simulator/illumination.py` 输出场包络：

`A_env = sqrt(I_inc / I0)`

`E_env_complex = A_env · exp(i phi_beam)`

当前默认：

`illumination_mode="overfill"`

它把横向 `x/z` 照明近似压平，但保留沿 `y` 方向穿过焦区的 finite transit window。

照明还会乘偏振投影因子：

`A_env = A_env_scalar · amplitude_factor_ill`

默认：

`illumination_polarization_mode="match_scattering"`

`cross_polarization_leakage=0.05`

### 4.6 事件级散射场

`nodi_simulator/scattering_trace.py` 的主式是：

`E_sca(t) = E_env_complex(t) · E_sca_unit_normalized · f_coupling(t) · exp(i phi_extra(t))`

默认耦合：

`coupling_model="gaussian_xy"`

`f_coupling = exp(-(x/wcx)^2 - (z/wcz)^2)`

默认相位模型：

`phase_model="relative_surrogate"`

`path_opd_model="single_pass"`

可理解为：

`phi_extra(t) = phi_path_x(t) + phi_path_z(t) + phi_focus_crossing(t)`

材料和投影相位来自 `E_sca_unit_normalized` 的复数相位。

### 4.7 干涉、噪声与读出

干涉层：

`E_det(t) = E_ref(t) + E_sca(t)`

`I_det(t) = |E_det(t)|^2`

`signal_trace(t) = I_det(t) - |E_ref(t)|^2`

展开：

`signal_trace = |E_sca|^2 + 2 Re(E_ref E_sca*)`

当前默认还允许 joint overlap 修正：

`cross_joint = 2 Re(overlap_factor · E_ref · E_sca*)`

默认：

`interference_overlap_mode="joint_overlap_integrated"`

噪声顺序是：

`raw interference -> pre-readout noise -> readout -> post-readout noise`

pre-readout 噪声包括：

- Gaussian noise
- shot-noise surrogate
- drift surrogate

默认读出：

`readout_model="lockin_surrogate"`

它生成 POD / NODI 两路，再选择 NODI lane 的 observable 作为：

`signal_detect`

当前默认：

`readout_observable_mode="in_phase"`

因此进入阈值层的是 NODI in-phase 通道。若切到 `magnitude`，则进入阈值层的是 NODI magnitude。

### 4.8 阈值与 pulse 提取

阈值只从前 20% 背景段估计：

`bg = signal_post_readout[:n_bg]`

`median = med(bg)`

`MAD = med(|bg - median|)`

`robust_std = 1.4826 · MAD`

`threshold = median + threshold_sigma · robust_std`

脉冲提取基于 `scipy.signal.find_peaks`，主要约束是：

- `height >= threshold`
- `width >= min_peak_width`
- `distance >= min_peak_interval`

当前默认：

`pulse_detection_mode="absolute"`

所以负脉冲也可被计为有效响应，但极性会保存在 `peak_signed_height / peak_polarity / phase_flip_fraction` 等字段里。

## 5. Batch 汇总、评分和 Gate

### 5.1 单事件判定

每个 event 可能有多个 peak。进入 batch 统计时，当前决策 basis 只取最强峰：

`best_peak = argmax(peak_height)`

所以：

- `detection_rate` 表示一个 event 是否至少检出一次。
- `mean_peak_height` 表示已检出 event 中最强峰的平均高度。
- 两者不是同一个物理量。

### 5.2 Batch 统计

核心输出包括：

- `n_detected`
- `detection_rate`
- `detection_rate_wilson_lb`
- `stable_detection_rate`
- `stable_detection_rate_wilson_lb`
- `mean_peak_height`
- `std_peak_height`
- `robust_cv_peak_height`
- `mean_peak_margin_z`
- `mean_local_snr`
- `phase_flip_fraction`
- `phase_flip_fraction_wilson_ub`
- `event_artifact_risk_score`
- `hit_rate_at_fixed_false_alarm`
- `roc_auc_event_vs_background`
- `d_prime_event_vs_background`

paired channel 还会输出：

- `single_channel_detection_rate(_wilson_lb)`
- `paired_channel_detection_rate(_wilson_lb)`
- `strict_paired_detection_rate(_wilson_lb)`
- `best_peak_pairing_rate`

### 5.3 all-crossing 与 selected-annulus 双口径

默认工程口径仍是 all-crossing：

`all_crossing_events = {i | event i crosses the simulated detection zone}`

`detection_rate = all_crossing_detection_rate = n_detected(all_crossing_events) / n(all_crossing_events)`

这个口径进入 `compute_case_score`、engineering gate、主 ranking 和正式 all-crossing 报告。

selected-annulus 是并行分析 lens，不替代主口径。它只在 batch summary 中对同一批 event 重新取一个条件分母：

`edge_norm_i = max(|x_norm_i|, |z_norm_i|)`

`selected_annulus_events = {i | a_min <= edge_norm_i <= a_max}`

当前默认配置写在 `SimulationConfig.selected_annulus_edge_norm_min/max`，默认 `a_min = 0.5`、`a_max = 0.8`，并校验 `0 <= a_min < a_max <= 1`。summary 输出：

- `selected_detector_mode_annulus_edge_norm_min`
- `selected_detector_mode_annulus_edge_norm_max`
- `selected_detector_mode_annulus_n_events`
- `selected_detector_mode_annulus_fraction`
- `selected_detector_mode_annulus_detection_rate`
- `selected_detector_mode_annulus_detection_rate_wilson_lb`
- `selected_detector_mode_annulus_mean_edge_norm`

如果 selected-annulus 分母为空，`n_events = 0`、`fraction = 0`，conditional detection rate、Wilson LB 和 mean edge norm 输出为 `NaN`，表示“该条件口径不可用”，不是 0% 检出。下游 route analysis 在这种情况下会保留 all-crossing 输出，并把 selected-annulus lens 标为 unavailable。

元数据治理：

- selected-annulus paper-fit 的 claim level 统一为 `paper_aligned_2022_nodi_proxy_lens`。
- paper alignment target 统一为 `tsuyama_2022_nodi_table_s1`。
- 这两个枚举值集中定义在 `nodi_simulator/design_claim_governance.py`，dashboard / joint-fit / sensitivity 输出都引用同一 registry，避免 `geometry`、`table_s1`、`geometry_table_s1` 这类裸字符串漂移。
- `paper_alignment_target` metadata 现在会校验 NODI 2022 readout 层约束：lock-in 约 `3000 Hz`、threshold `5/10 sigma`、minimum peak width 约 `2.5 ms`、`readout_observable_mode="magnitude"`、`pulse_detection_mode="positive"`。不满足这些字段的 raw row 不能被标为同一个 `tsuyama_2022_nodi_table_s1` paper-alignment target。
- selected-annulus claim compatibility 现在会比较 `paper_alignment_target`、selected-annulus source、edge-norm min/max，以及 engineering-relative 场景下的 schema / analysis lane；这些字段不一致时应 fail fast，而不是把不同口径结果拼成同一个 paper claim。
- joint-fit 中的 `MIN_ANNULUS_FRACTION = 0.25` 是 selected-annulus paper-fit 质量 guardrail：annulus 子集太小的 candidate 不能被标成 paper-fit 通过。它不是 Tsuyama 原文参数；若改变，需要跟 annulus ratio sensitivity 决策一起记录。
- `joint_fit_score` 是 lower-is-better 的 loss-style penalty，由 selected-rate、signal-ratio、size-exponent、SNR-ratio、transfer/size regularization 和 guardrail penalty 共同组成；不能按“分数越高越好”解释。
- 当前 selected-annulus joint-fit / sensitivity 的粒径诊断口径由代码常量固定为 Au `20/30/40/60 nm`、Ag `40/60 nm`。不要把外部 review 中未经当前论文/代码复核的 Au `30/50/100 nm` 口径写入该 lane。
- `tools/audits/tsuyama_selected_annulus_joint_fit.py` 只允许 paper-target-compatible variants 进入 paper-aligned joint fit。`in_phase + absolute` 这类诊断 variant 不满足当前 `tsuyama_2022_nodi_table_s1` 的 `magnitude + positive` metadata guard，会在 candidate 构建阶段 fail fast，而不是先跑完 sweep 再失败。

paper target 边界：

- all-crossing 主口径不对齐 Tsuyama paper detection-rate target。它只用于工程 gate、主 ranking，以及监督 selected-annulus 子集相对 all-crossing 的变化。
- selected-annulus paper-audit lane 当前使用代码中的 2022 NODI surrogate detection-rate target：Au20 `0.30`、Au30 `0.60`、Au40 `0.78`、Au60 `0.92`，并同时检查 Ag/Au signal ratio、Au size exponent 和 Au30/Au20 SNR ratio。
- 这些 detection-rate targets 是当前 paper-audit surrogate / operational targets，不是 Tsuyama 原文直接给出的 crossing-to-detection efficiency table。Phase 2 参数反演前必须先做 target audit，把每个 target 标成 `direct / inferred / operational / diagnostic-only`，并记录论文来源、机制和是否可进入 hard acceptance。
- `tools/audits/tsuyama_detection_rate_calibration.py` 中较早的 Au20 `0.15`、Au30 `0.42`、Au40 `0.60`、Au60 `0.70` bands 只属于 legacy screening / broad operational calibration lane，不能和当前 selected-annulus joint-fit targets 混合作为 Phase 2 主验收。
- `tools/audits/tsuyama_2022_classification_lane.py` 的 `paper_target_accuracy = "71.9 +/- 4.0%"` 在完成 source/protocol audit 前只作为 diagnostic metadata；classification / peak-ratio lane 不进入 primary joint-fit pass/fail，也不能驱动全局参数反演。
- 2020 POD thermal counting 中近 100% Au20 detection 是另一种热读出机制，不是当前 NODI selected-annulus paper-audit target；不得用 NODI all-crossing 或 selected-annulus 结果去硬对齐这条 100% 结论。

lane 分层：

```text
2022 NODI paper-audit lane
  JOINT_CASES: 488/532/660 nm x (800,550)/(1200,550) nm
  particles: Au 20/30/40/60 nm + Ag 40/60 nm
  purpose: annulus / readout / transfer / size-response surrogate calibration
        |
        | only after a signed decision and metadata/claim guard pass
        v
EV engineering main-library lane
  ev_design grid: 404/488/532/660 nm x full W/H grid x engineering particle profile
  purpose: all-crossing primary ranking + selected-annulus cross-check
```

Phase 2 参数反演只能在 paper-audit lane 上做；Phase 3 才能把收敛出的 proxy 配置推广到工程主库 lane。报告里凡是提到 `28` 粒径全网格或 Au/Ag `4+2` 粒径 paper audit，必须显式标明属于哪一层。

Phase 2 启动顺序：

1. 先做 paper target audit，而不是直接调参；没有 source anchor 或只适合作 diagnostic 的 target 不进入 hard acceptance。
2. 固定 paper identity：JOINT_CASES、Au/Ag 粒径集合、`0.5-0.8` annulus、`magnitude + positive` readout、threshold 仅 `{5,10}`、minimum peak width 仅 `2.0-3.0 ms`。
3. 优先反演 readout/noise/threshold/blank calibration，再看 transit/flow/beam-waist event-shape，最后才看 bounded silver transfer 与 bounded Au size-response correction。
4. Phase 2 acceptance 必须先过 hard guardrails：target metadata / claim compatibility、annulus fraction、transfer gain、size-response delta、rho/reference/NA 状态、no all-crossing paper target、no global EV default mutation。数值 criteria 只在 guardrails 全过后评估。
5. Classification lane 只做 sanity diagnostic：报告 usable rows、SVM configuration 和 simulated accuracy，但不作为 paper-fit 主门槛。

物理解释边界：

- 它是初始粒子位置的事后空间过滤，近似 Tsuyama 2022 NODI 的有效采样区语义。
- 它不是收集端 NA 环带，不替代 detector ROI、BFP mask、collection operator 或 NA cutoff。
- 它适用于 2022 NODI paper-aligned comparison 和 EV route cross-check；不适用于把 2020 POD thermal counting 的近 100% Au20 检出率硬解释成 NODI selected-annulus detection。
- `0.5-0.8` 是当前 canonical default；若全量重算前要改变它，应先跑 2022 NODI 几何上的 annulus ratio sensitivity，并把决策写入结果和文档。2026-05-02 已完成 smoke、7-window medium 与 top-window 1000-event focused sensitivity：medium 中 `0.6-0.9` 有小幅优势，但 focused run 中 paper-fit candidate 仍以 `0.5-0.8` 最低 penalty，且 effect/方向不满足 canonical-change rule；因此当前不改变 default。

### 5.4 Summary-only accumulator

正式 precompute 不保存每个 event 的完整 trace，而是保存 summary 统计和必要 diagnostics：

`_BatchSummaryAccumulator.finalize() -> summary`

这保证全量 `fine` 重算不会把磁盘和内存压爆。若需要完整 trace，应在 inspector / single-case 路径按需重算。

### 5.5 评分

传统探索分数：

`score = w_height · H_norm + w_rate · R_norm - w_cv · CV_norm`

它适合在同一 sweep 内找候选区域，不是跨任意 sweep 的绝对标尺。

工程分数：

`engineering_score`

`= 1.0 · stable_rate_norm`

`+ 0.7 · threshold_margin_norm`

`+ 0.4 · local_snr_norm`

`+ 0.5 · auc_norm`

`+ 0.5 · hit_rate_norm`

`+ 0.3 · d_prime_norm`

`- 0.4 · CV_norm`

`- 0.4 · robust_CV_norm`

`- 0.5 · phase_flip_penalty`

`- 0.6 · event_artifact_risk_norm`

### 5.6 工程 Gate

hard gate 在连续评分之外工作。当前默认检查：

- `engineering_min_detected_events = 5`
- `engineering_min_detected_fraction = 0.10`
- `engineering_min_stable_detection_rate = 0.20`
- `engineering_max_phase_flip_fraction = 0.50`
- `engineering_min_mean_peak_margin_z = 0.50`

默认 basis：

`engineering_decision_basis="final_decision"`

最终工程分数：

若 gate 通过：

`final_engineering_score = engineering_score`

若 gate 未通过：

`final_engineering_score = -10 - engineering_gate_failed_count + engineering_score`

排序意图是：

1. 先分是否通过 gate。
2. 再分失败项数量。
3. 最后比较连续 `engineering_score`。

## 6. Governed Provenance 与解释边界

当前输出链会同步记录 claim level、calibration state、单位约定、reference route、readout route、selection function、event QC、EV reporting、count/population scaffold 等字段。这些字段不是额外“物理求解器”，而是结果解释边界。

### 6.1 Calibration state

`utils.build_calibration_state_diagnostics(...)` 汇总：

- reference calibration
- scattering / standard particle calibration
- detector unit chain
- readout route
- count model
- BFP ROI mask contract
- synthetic fixture / manifest kind 状态

并输出：

- `reference_calibration_level`
- `scattering_calibration_level`
- `detector_calibration_level`
- `readout_calibration_level`
- `count_calibration_level`
- `output_claim_level`

没有真实 measured calibration 时，`output_claim_level` 不会升级成 calibrated quantitative。

### 6.2 Detector-unit boundary

当前相对场不会直接换算成 photon units 或电压。缺口包括：

- incident power density
- `dCsca/dΩ -> dP_sca/dΩ`
- detector etendue / absolute throughput
- photodiode responsivity
- transimpedance gain
- ADC conversion
- lock-in voltage gain

因此 baseline-normalized `E_sca` 是 relative diagnostic，不能直接当 detector-unit field。

### 6.3 Count / population boundary

默认：

`count_prediction_model="not_applied"`

所以 `detection_rate` 表示“给定一个事件发生后是否被检出”，不是实验计数率、粒子浓度或绝对 LOD。

count likelihood、population inference、EV population prior、OOD detection 当前用于 scaffold 和风险提示，不把 per-event detectability 自动升级为绝对浓度结论。

### 6.4 Design claim

`nodi_simulator/design_claim_governance.py` 与 `nodi_simulator/design_postprocess.py` 会把当前 case 归纳成可审计 design claim：

- `minimum_output_schema_status`
- `relative_design_eligible`
- `within_lambda_design_eligible`
- `absolute_global_green_eligible`
- `final_green_eligible`
- `final_recommendation_band`
- `primary_blocker_summary`
- `design_recommendation_status`

无实测闭环时，`final_green_eligible=False` 是预期的保守结论，不代表计算失败。工程分析应区分三层 eligibility：

- `relative_design_eligible`
  当前 proxy/relative 模型内是否可作为工程排序候选。它要求工程 gate、readout semantics、detector operator、double-count guard、recompute manifest、event QC、selection function 与 EV integrity 这些内部一致性门通过，但不要求 detector-unit / absolute calibration 闭环。
- `within_lambda_design_eligible`
  是否可在同一波长内比较 W/H 几何。
- `absolute_global_green_eligible`
  是否可做跨波长、detector-unit 或 calibrated global green claim；当前无实测数据时通常为 `False`。

允许的说法是：

- 当前模型口径下的相对 design ranking
- 工程风险和 blocker
- 下一步需要补什么标定或实测

不允许的说法是：

- calibrated SNR
- absolute LOD
- absolute EV concentration
- 脱离当前 sweep 的跨系统绝对排序
- 生物 EV specificity 结论

## 7. Event Block 与性能路径

正式 precompute 默认走 scalar event loop，而不是 summary-only 事件块路径：

- `vectorized_event_engine="off"`
- `event_block_size=32`
- `event_block_rng_order="event_loop_order"`

`off` 不代表低精度模式，而是逐 event 运行最保守的基准事件循环。它保留当前结果语义，并避免 16C/32T 全量重算中 `event_block_v3` 实测出现的 block 调度和多线程叠加开销。

`event_block_v3` 会把 trajectory / illumination / scattering / interference / readout / threshold / peak summary 中可批量处理的部分放进 block，同时保持最终输出字段和 scalar summary 语义一致。它仍可通过 CLI 显式启用，用于性能实验或回归对照，但不是当前正式全量结果库默认。

`event_loop_order` 是正式默认，因为它保持 scalar event loop 的随机数消费顺序。`block_lane_order` 会改变个体 event 轨迹，只能用于显式实验，不是当前正式结果库口径。

若某个 case 的模式不满足 block fast path 条件，代码会记录 fallback reason，而不是悄悄改变结果含义。

## 10. 最简公式链

### Step 1：本征散射

`x = 2π n_m a / λ0`

`m_rel = ñ_p / n_m`

`Mie -> S1(θ), S2(θ), Csca, Cext, Cabs`

`Esca_unit_amp(θ) = sqrt((|S1|^2 + |S2|^2) / (2 k_m^2))`

### Step 2：探测算子

`field_sca(θ) = Esca_unit_amp 或 S2/k_m 或 S1/k_m`

`E_sca_detected_complex = L_det[field_sca]`

`E_sca_unit_normalized = E_sca_detected_complex / E_sca_ref(λ)`

### Step 3：参考场

`E_ref = reference_model(W, H, λ, n_m, rho)`

route 决定它属于 calibrated、paper-aligned comparison、engineering fallback 还是 legacy debug。

### Step 4：事件级散射场

`E_sca(t) = E_env_complex(t) · E_sca_unit_normalized · f_coupling(t) · exp(i phi_extra(t))`

### Step 5：干涉和读出

`signal_trace(t) = |E_ref + E_sca(t)|^2 - |E_ref|^2`

`signal_post_readout = post_noise(readout(pre_noise(signal_trace)))`

### Step 6：阈值和 pulse

`threshold = median(bg) + threshold_sigma · 1.4826 · MAD(bg)`

`features = find_peaks(signal_post_readout 或 |signal_post_readout|)`

### Step 7：batch 和排序

`summary = summarize_batch(events)`

`score = f(mean_peak_height, detection_rate, CV)`

`engineering_score = f(stable_rate, margin, local_snr, auc, hit_rate, d', cv, phase_flip, event_artifact_risk)`

`final_engineering_score = gate_then_rank(engineering_score)`

## 11. 使用边界

适合用它做：

- 比较不同粒子、波长、通道几何的相对检测趋势。
- 解释某个设计为什么更容易或更难检出。
- 在实验前做参数空间粗筛。
- 发现当前模型需要补的标定、对照或实验约束。

不适合直接把它当成：

- 绝对信号强度真值预测器。
- 完整实验系统的严格数值复现。
- 绝对 EV 浓度或 LOD 计算器。
- 脱离当前 sweep 语义的通用排序常数。

最准确的定位是：

**一套按“本征散射 -> 观测算子 -> 干涉读出 -> 统计判定 -> 工程筛选 -> 可审计结果库”分层实现的 NODI 工程模拟主链。**
