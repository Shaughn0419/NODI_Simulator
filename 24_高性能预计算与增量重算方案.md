## 2. 正式任务

正式重算目标：

- total cases: `32032`
- events/case: `10000`
- total events: `320320000`
- particles: Au20/Au30 anchors + gold `40-300 nm` + biomimetic EV/sEV exosome-like particles `40-300 nm`
- wavelengths: `404 / 488 / 532 / 660 nm`
- channel grid:
  - `W = 500 / 600 / 700 / 800 / 900 / 1000 / 1100 / 1200 / 1300 / 1400 / 1500 nm`
  - `H = 500 / 550 / 600 / 650 / 700 / 800 / 900 / 1000 / 1100 / 1200 / 1300 / 1400 / 1500 nm`

结果定位：

- 可以用于 relative/proxy EV/NODI engineering design ranking。
- 可以用于同一波长内 W/H 相对选型、Au20/Au30 anchor 风险解释、EV ensemble worst-case detectability、detector/operator blocker 分析。
- 不能宣称 absolute SNR、absolute LOD、真实 EV 浓度、真实跨波长最优、真实跨 objective 最优或 biological exosome-specific claim。

## 3. 正式命令

优先在正式计算机器上运行。若是 28-worker 目标机器，用下面命令原样执行：

```bash
source .venv/bin/activate

export NUMBA_NUM_THREADS=4

PYTHONPATH=. python -m dashboard.precompute \
  --grid ev_design \
  --particle-profile full_range_biomimetic_exosome_with_anchors \
  --tag full_range_biomimetic_exosome_with_anchors_10000e \
  --workers 28 \
  --vectorized-event-engine off \
  --event-block-size 32 \
  --event-block-rng-order event_loop_order \
  --freeze-probe-report \
  --artifact-profile standard \
  --progress-interval 2 \
  --resume \
  --checkpoint \
  --checkpoint-batch-size 100 \
  --checkpoint-flush-interval 5 \
  --output results
```

如果目标是 EV optical-uncertainty ensemble 主库，把命令中的 profile/tag 改为：

```bash
  --particle-profile ev_design_biomimetic_ensemble_with_anchors \
  --tag ev_design_biomimetic_ensemble_with_anchors_10000e \
```

Windows PowerShell 等价写法：

```powershell
$env:NUMBA_NUM_THREADS="4"
```

如果只在当前 10-core 本机运行，可以把 `--workers 28` 改成 `--workers 8`，但不要改：

- `--grid`
- `--particle-profile`
- `--tag`
- `--artifact-profile`
- `--vectorized-event-engine`
- `--event-block-size`
- `--event-block-rng-order`

不要加 `--allow-partial`。case 失败必须让重算失败，而不是保存偏置的 partial database。

如果中断，原命令重跑即可。`--resume --checkpoint` 会复用已完成 case 并补跑缺失 case。

恢复前会校验 checkpoint manifest 的 `grid / config_tag / particle_profile / total_cases` 是否与本次运行一致；如果 checkpoint 身份不匹配，或存在无法识别身份的 checkpoint chunk，程序会拒绝 resume，避免把不同 profile/grid 的 raw cases 混入正式库。

如果出现 failed cases：

1. 不要改成 `--allow-partial`。
2. 先读失败 case 的异常和 checkpoint manifest。
3. 修根因后用同一正式命令 resume。
