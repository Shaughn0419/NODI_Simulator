## 1. 为什么单开这份文档

我更建议把“完整全波理论”单开，而不是继续并进 [25_核心计算逻辑与公式总说明.md](./25_核心计算逻辑与公式总说明.md)。

原因不是它不重要，而是它和文件 25 的职责不同：

- 文件 25 负责回答：当前工程到底怎么算、为什么这样分层、默认主链怎么理解
- 本文负责回答：如果完全按更严格的电磁前向问题来写，这个问题应该长什么样；当前实现相对它省略了哪些层次

如果把两者强行并在一起，会立刻出现两个问题：

1. 主链说明会失焦。读者会分不清“当前已经实现的逻辑”和“理想上应该进一步做的逻辑”。
2. 审查口径会混乱。工程说明需要可落地、可对照代码；全波说明则需要交代理论上限和简化边界。

所以更清楚的组织方式是：

- 文件 25：当前工程主链
- 本文：理论上限与边界

## 2. 这里说的“完整全波理论”到底指什么

这里的“完整全波”不是一句“用 Maxwell 方程求一下”就结束，而是指下面这条完整前向问题：

`入射场 + 纳米通道结构 + 介质 + 粒子 + 边界条件 + 收集光学系统 + 探测器读出`

共同决定最终的时域检测信号。

在频域单色近似下，电磁场满足：

\[
\nabla \times \mathbf{E} = -i\omega\mu_0 \mathbf{H},
\qquad
\nabla \times \mathbf{H} = i\omega\epsilon_0 \epsilon_r(\mathbf{r},\omega)\mathbf{E} + \mathbf{J}
\]

消去磁场后，可写成矢量 Helmholtz 形式：

\[
\nabla \times \mu_r^{-1}(\mathbf{r}) \nabla \times \mathbf{E}
- k_0^2 \epsilon_r(\mathbf{r},\omega)\mathbf{E}
= i\omega\mu_0 \mathbf{J}
\]

这里真正困难的地方在于：`ε_r(r,ω)` 不是一个均匀常数，而是同时包含了

- 液体介质
- 通道壁材料
- 粒子本身
- 可能的空间非均匀照明与边界

所以严格来说，这不是“自由空间里一颗球”的散射问题，而是“结构背景中的局域散射体”问题。

## 3. 理想的完整前向链应该怎么写

如果要把这个问题写成严格的全波前向链，比较清楚的分法是下面 6 层。

### 3.1 背景结构场

先去掉粒子，只保留：

- 通道几何 `W, H`
- 壁材料 `n_wall(λ)`
- 介质 `n_medium(λ)`
- 入射光束
- 边界条件

此时应先解出“无粒子时”的背景场：

\[
\mathbf{E}_{\mathrm{bg}}(\mathbf{r}; W,H,\lambda,\text{illumination})
\]

这一步本质上对应：

- blank channel 的 reference 场
- 通道结构对入射场的调制
- 真实的空间相位与振幅分布

在严格做法里，reference 不是一个经验 `rho`，也不是一个单一 `g(W,H,\lambda)`，而是背景结构在探测链中投影后的结果。

### 3.2 结构背景中的粒子散射

加入粒子后，严格问题不再是“均匀介质里的标准 Mie”，而是：

\[
\epsilon_r(\mathbf{r}) = \epsilon_{r,\mathrm{bg}}(\mathbf{r}) + \Delta\epsilon_{r,\mathrm{particle}}(\mathbf{r}; \mathbf{r}_p)
\]

严格场可以写成体积分形式：

\[
\mathbf{E}(\mathbf{r})
=
\mathbf{E}_{\mathrm{bg}}(\mathbf{r})
+
k_0^2 \int \mathbf{G}_{\mathrm{bg}}(\mathbf{r},\mathbf{r}')
\Delta\epsilon_{r,\mathrm{particle}}(\mathbf{r}')
\mathbf{E}(\mathbf{r}')
\mathrm{d}^3\mathbf{r}'
\]

这里的关键不是公式形式本身，而是：

- Green 张量不再是均匀介质 Green 函数，而是结构背景 `G_bg`
- 粒子散射会和壁、通道、偏振、局部背景场发生耦合

这也是为什么“严格全波”比当前 `Mie + geometry surrogate` 难得多。

### 3.3 光学收集与成像算子

完整全波并不在“得到远场散射”后结束，还需要经过收集与成像系统。

严格来说，探测端应当是一个矢量收集算子：

\[
\mathbf{E}_{\mathrm{det}}(\mathbf{x}_{\mathrm{sensor}})
=
\iint_{\mathrm{pupil}}
\mathbf{P}(u,v)\,
\mathbf{E}_{\mathrm{exit}}(u,v)\,
e^{ik \Phi(u,v;\mathbf{x}_{\mathrm{sensor}})}
\mathrm{d}u\,\mathrm{d}v
\]

这里会同时包含：

- objective / pupil 的有限孔径
- 偏振投影
- slit / pinhole
- 像差与相位项
- 不同空间频率的传递

所以严格的“检测角”并不是一个 `theta_det`，而是一个真正的矢量 pupil integral。

### 3.4 干涉检测

传到探测面后，真正的光学观测量仍然不是“散射场幅值”本身，而是与背景/reference 共同形成的检测强度：

\[
I(\mathbf{x}, t)
=
\left|
\mathbf{E}_{\mathrm{ref}}(\mathbf{x}, t)
+
\mathbf{E}_{\mathrm{sca}}(\mathbf{x}, t)
\right|^2
\]

如果做空间积分或像素积分，则进一步变成：

\[
S(t) = \int_{\mathrm{pixel}} I(\mathbf{x}, t)\,\mathrm{d}A
\]

此时交叉项：

\[
2\operatorname{Re}\left[\mathbf{E}_{\mathrm{ref}}\cdot \mathbf{E}_{\mathrm{sca}}^*\right]
\]

是否足够强、是否相位稳定、是否被投影到同一偏振基底上，都会直接影响可检测性。

### 3.5 运动、时间采样与电子读出

完整前向问题还必须把粒子轨迹接进去：

\[
\mathbf{r}_p = \mathbf{r}_p(t)
\]

于是严格场和信号都变成时变量：

\[
S(t) = \mathcal{D}
\left[
\mathbf{E}_{\mathrm{bg}}(\mathbf{r}; \mathbf{r}_p(t))
,
\mathbf{E}_{\mathrm{sca}}(\mathbf{r}; \mathbf{r}_p(t))
\right]
\]

其中 `𝒟` 不只是“看一眼信号”，而应当包含：

- 探测器积分时间
- 带宽
- 锁相或调制读出
- shot noise / read noise / drift
- 阈值与事件判别

这一步说明：严格物理前向问题和最终“可检测事件”之间，还隔着一个完整的观测链。

### 3.6 batch 统计与判别

即便你真的做完了单事件全波前向问题，也还没有直接得到“哪个设计更好”。

因为工程决策用的是：

- 检出率
- 稳定性
- 与背景的可分离度
- 对随机位置、扩散、速度波动的鲁棒性

所以排序最终仍然要落回统计层，只是这时统计层的输入会比当前 surrogate 主链更物理。

## 4. 为什么当前工程不直接做完整全波

不是因为完整全波不正确，而是因为在这个项目的目标下，它并不是当前最合适的主实现。

当前不直接做完整全波，主要有 5 个原因：

### 4.1 参数扫描规模过大

当前工程要比较的是大量 `(W, H, λ, particle, noise, trajectory)` 组合。

如果每个 case 都做一遍严格结构电磁求解，再叠加时域轨迹和 batch 统计，那么离线全量主库几乎不可维护。

### 4.2 当前问题更关心“工程排序”，而不是单点极限精度

这个项目的主要任务是：

- 找设计趋势
- 找候选区
- 解释为什么某些组合更可能被检出

这更适合用“可解释的分层 surrogate 链”，而不是直接把一切都埋进黑箱数值求解器。

### 4.3 需要可审计的中间量

当前 dashboard 之所以能讲清楚：

- 本征散射强不强
- reference 来自哪里
- 哪一层让信号变强或变弱
- 为什么最后检出率变了

靠的是中间量都还能单独看。

如果直接换成一次性全波黑箱求解，中间机制反而更难展示。

### 4.4 需要支持增量重算

当前工程已经按“物理层级”拆开，所以：

- 改噪声，不重算 Mie
- 改评分，不重算 reference
- 改 reference fallback，不重算 pulse extraction 以前的全部库

全波主链如果不再分层，会大幅削弱这种可维护性。

### 4.5 还缺少足够的实验标定闭环

即便投入很大代价做了更完整的数值电磁求解，如果没有：

- 空通道标定
- pupil / slit / pinhole 实际参数
- 相位与偏振校准
- 读出链标定

最终仍然未必比“标定表 + physics-informed surrogate”更可靠。

## 5. 当前 surrogate 主链和理想全波的对应关系

下面这张表最适合回答“当前到底省略了什么”。

| 理想全波对象 | 当前工程对应 | 当前这样做的原因 | 当前损失的东西 |
| --- | --- | --- | --- |
| 结构背景场 `E_bg(r)` | `calibrated_lookup` / `channel_angular_surrogate` / `geometry_scaled` | 保留几何趋势、reference 量级和相位审计 | 没有真正求出结构内空间场 |
| 结构背景中的粒子散射 | 均匀介质 `Mie` + 几何/相位 surrogate | 先把粒子本征散射和系统效应拆开 | 没有显式求 wall-particle 多重散射 |
| 矢量 pupil integral | `pupil_slit_surrogate` | 低成本表达有限角收集、phi/slit/pinhole 与偏振投影 | 不是严格成像理论 |
| 完整时变 reference 场 | `compute_reference_field_trace` + `reference_spatial_mode` + `phi_ref` diagnostics | 让 reference 至少随几何与局部位置变化 | 不是全场演化 |
| 完整电学读出 | `lockin_surrogate` + pre/post-readout noise | 保留最关键的带宽与读出层语义 | 不是频谱级电子学模型 |
| 实验判别器 | `pulse_detection + batch + engineering_score` | 直接服务工程排序 | 仍是工程判据，不是最终实验判读真值 |

这里还需要补一句，避免把“surrogate”误读成“只有很粗糙的经验幂律”：

当前实现里已经有几层是明显在往更严格观测链靠拢的，只是还没有到完整全波求解。

- `channel_angular_surrogate`
  已经不再只是单个 `g(W,H,λ)` 幂律，而是先构造通道角谱 surrogate，再用共享探测算子 collapse。
- `pupil_slit_surrogate`
  已经不再只是“取一个检测角”，而是把 `theta / phi / pupil / slit / pinhole / polarization projection` 压进一个统一 operator。
- `reference_spatial_mode="cross_section_surrogate"`
  已经让 reference 从纯 case-level 标量推进到 event-level trace。
- `interference_overlap_mode="joint_overlap_integrated"`
  已经在交叉项里显式保留了由共享探测算子导出的 overlap 语义。

所以更准确的说法不是“当前实现完全没有接近全波的结构”，而是：

**当前已经把若干最关键的观测链语义显式拆出来了，但这些语义目前仍由可审计的 surrogate 表达，而不是由结构电磁全波求解统一生成。**

## 6. 当前模型什么时候是合理的

当前 surrogate 主链最适合下面这些问题：

1. 我想比较不同粒子、波长、通道几何的相对检出趋势
2. 我想知道是 `Mie / reference / phase / noise / threshold` 的哪一层在主导结果
3. 我需要大规模 sweep、批量排序、dashboard 展示和增量重算
4. 我关心的是“工程上哪个区更值得做实验”，而不是“某个单点的绝对场值是否达到论文级精度”

如果问题落在这些范围内，当前主链是合理的。

## 7. 哪些问题必须认真考虑全波升级

一旦问题落到下面这些区域，就不应再把当前 surrogate 视作足够：

1. 你要做绝对强度或绝对相位的定量预测，而不是相对排序
2. 通道尺寸已经进入 `W ~ λ_eff`、`H ~ λ_eff` 的强结构电磁区
3. 偏振、像差、pupil 相位、收集路径对结论高度敏感
4. 粒子和壁的近场耦合、多重散射、模态耦合可能主导结果
5. 你需要解释与 blank-channel 标定、空间场分布、成像面结构强相关的细节
6. 你准备把结果直接拿去做严格实验反演，而不是设计筛选

这时更合理的做法不是继续往 surrogate 上补 patch，而是显式进入更高保真度的求解链。

## 8. 如果以后往全波方向推进，建议怎么做

如果未来真要往“更完整的全波链”推进，我建议仍然分阶段推进，而不是一次性替换整个工程。

### 8.1 第一阶段：先升级 background / reference，不先替换整条链

优先级最高的是：

- blank channel 背景场
- reference 的空间分布
- pupil / slit / pinhole 的真实传递

原因是它们最直接决定干涉项，而且最能和实验标定闭环。

### 8.2 第二阶段：再升级粒子在结构背景中的散射

这一层才是真正昂贵的部分。可以考虑：

- 用结构背景 Green 张量做局域修正
- 对有限子空间做更高保真度数值库
- 再把高保真库压回 surrogate / lookup

### 8.3 第三阶段：保留 batch 与工程评分层

即便前向求解更物理，`事件 -> batch -> engineering decision` 这条链仍然应该保留。

因为实验决策本来就不是“哪一个瞬时场值最大”，而是“哪一个设计在随机事件里更稳定、更可检出”。

## 9. 给当前读者的最直接结论

如果你现在主要是在维护这套工程或读 dashboard，我的建议是：

- 想理解当前怎么算：看 [25_核心计算逻辑与公式总说明.md](./25_核心计算逻辑与公式总说明.md)
- 想理解为什么当前不能把 surrogate 当全波真值：看本文
- 想继续升级实现：优先从 `reference / collection operator / calibration bridge` 往前推，而不是先推翻整条 batch 主链

**完整全波理论应该单开说明；文件 25 负责当前主链，本文负责理论上限、缺口和升级方向。**
