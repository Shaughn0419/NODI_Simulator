## 总结论

当前主链已经从单一 engineering surrogate 升级为明确分层的 governed pipeline：

- `engineering_ranking`
- `paper_aligned_comparison`
- `reference_calibrated_relative`
- 更高等级的 calibrated / photon-unit / experimental count-rate claim 只有在真实数据和高保真闭环齐备后才能解锁

最新复核后的更准确定位是：

> **工程排序主链 + Tsuyama 对齐诊断 + 定量闭环边界声明。**

当前代码可以用于无实测数据下的工程探索、相对排序、paper-aligned qualitative / normalized comparison，以及明确降级的 nominal prediction。它还不能被称为已经完成的 Tsuyama-aligned quantitative model。除全量重算外，P0-P6 的代码/测试层 hard gate 已完成；这些 gate 的目的不是新增大模型复杂度，而是防止 active reference、joint overlap、baseline-normalized scattering、sampled lock-in 和材料库被过度解释成 absolute quantitative truth。

## 最新代码复核完成对照

本节原本是 P0-P6 修正路线；现在作为完成态对照保留。当前代码锚点如下：

- `reference_field.py` 已导出 `reference_solver_active_field_source`、`reference_solver_detector_bridge_status`、`paper_aligned_reference_claim`，并把未接入 active BFP detector bridge 的 paper route 降为 diagnostic-only。
- active angular surrogate 的 `abs(Δn)`、`phase_delay/2` 与 `reference_phase_grating_phase_rad` 已被标注为 amplitude-envelope surrogate；signed phase / polarity claim 由 `theta_signed`、`thin_phase_complex_factor` 与 blocker 字段治理。
- `interferometric_trace.py` 在 `joint_overlap_integrated` 下导出 joint-overlap signal 与 scalar-surrogate status，避免把 legacy `E_det_complex` 误读为严格满足 `I=|E|^2` 的物理场。
- `parameter_sweep.py` 对 baseline-normalized scattering 输出 absolute route blockers；默认 `scattering_normalization_route="baseline_particle_relative"` 不允许 detector-unit / photon-unit claim。
- readout 链已有 sampled-carrier underresolved gate；Tsuyama 2024 paired preset 默认走 analytic lock-in surrogate / no-carrier-sampling 口径。
- `materials.py` 已有 Au/Ag tabulated visible material、water/PBS visible Cauchy nominal dispersion、fused_silica Sellmeier nominal dispersion，并输出 material DB coverage diagnostics。
- `detector_units.py`、`uncertainty.py`、`count_generation.py`、`interface_correction.py`、`photothermal_pod.py` 当前定位正确：它们是 boundary / claim-level 防过度宣称模块，不是已经闭合的 detector-unit / POD / full-wave 绝对模型。

### P0：Active reference source claim gate

状态：已完成为 hard gate。

已落地字段：

- `reference_solver_active_field_source`
- `reference_solver_detector_bridge_status`

当前验收口径：`paper_aligned_phase_filter` 未使用 `tsuyama_E_diffraction_BFP` 作为 active detector field 时，summary / metadata 显示 diagnostic-only；只有 active BFP bridge 或 `calibrated_lookup` 才能提升 reference claim。

### P1：Signed active reference phase gate

状态：已完成为 phase / polarity blocker。

engineering fallback 可继续使用 `abs(Δn)` 与 `phase_delay/2` 做 amplitude envelope，但必须输出 `phase_delay_over_2_role = amplitude_envelope_surrogate_only`、`polarity_claim_from_phase_delay_over_2_allowed = False`、`absolute_polarity_claim = blocked_active_reference_phase_surrogate`。paper / phase diagnostics 继续由 signed `theta_signed` 与 `thin_phase_complex_factor = exp(iθ)-1` 提供。

当前验收口径：小相位极限满足 `angle(exp(iθ)-1) -> sign(theta) * pi/2`；翻转 `(n_medium - n_wall)` 符号时，reference / POD polarity diagnostics 翻转或降级。

### P2：Joint overlap field semantics

状态：已完成为 field-semantics gate。

joint 模式下已导出 `E_det_complex_scalar_surrogate`、`I_det_joint_overlap`、`signal_trace_joint_overlap` 与 `E_det_complex_physical_status = scalar_surrogate_not_intensity_consistent`。legacy `E_det_complex` 保留为兼容 alias，不再作为 joint 模式下的严格物理场解释。

当前验收口径：`joint_overlap_integrated` 下 `I_det` 语义来自 joint overlap；`collapsed_then_multiplied` 下可继续保持原有 scalar `E_det_complex` 语义。

### P3：Absolute route blocker for baseline-normalized scattering

状态：已完成为 absolute-claim blocker。

当 `scattering_normalization_route == baseline_particle_relative` 时，输出只允许 engineering / relative 口径，`detector_unit_claim_allowed = False`、`photon_unit_claim_allowed = False`。synthetic `K_sca` 或 single-point residual scale 不会解锁 absolute quantitative。

未来若要提升，仍必须补齐 `dP_sca/dΩ = I_inc * dCsca/dΩ`、detector etendue / throughput、photodiode responsivity、TIA / ADC、lock-in gain、standard-particle `K_sca` 和 held-out validation。

### P4：Readout sampled-carrier anti-alias hard gate

状态：已完成为 readout validity gate。

sampled carrier route 现在要求 `fs_internal >= 10 * max(pod_lockin_frequency_Hz, nodi_lockin_frequency_Hz)`；不满足时输出 `readout_sampling_validity = carrier_underresolved` 并降级 claim。`tsuyama_2024_paired_*` preset 默认走 `analytic_lockin_surrogate` / `analytic_demod_no_carrier_sampling`，避免把低采样 event trace 误读为 raw photodiode carrier。

### P5：Material DB coverage and naming cleanup

状态：已完成为 material coverage / naming cleanup。

当前 `materials.py` 已包含 Au/Ag visible tabulated 数据、`water` visible Cauchy nominal surrogate、`pbs_1x` water Cauchy + nominal PBS offset、`fused_silica` Malitson Sellmeier nominal dispersion，并输出：

`intrinsic_scattering.py` 已新增 canonical `k_m` 字段；`k_m_inv` 只作为 backward-compatible legacy alias 保留，并明确不是 `1/k_m`。

### P6：Package-level completeness and smoke tests

状态：已完成为包级 smoke / regression gate。

当前固定验证包括 `ruff`、`pyright`、`mypy`、`pytest` 和排除虚拟环境 / macOS resource fork 的 project-source `compileall`。smoke / regression 覆盖 channel angular surrogate、paper-aligned phase filter、synthetic calibrated lookup、Tsuyama 2022 / 2024 preset、core-shell / EV-sEV family、claim level、synthetic fixture status、detector-unit boundary、POD boundary 与 count boundary。

synthetic fixture 不会解锁 calibrated quantitative claim。

### P7：EV/sEV、Mie、POD 红线继续保持

保留当前 claim 降级，不要为了“补完整”而过度承诺：

- EV/sEV 只能输出 `optical_EV_like_particle`，不能从光学散射宣称生物发生学 exosome。
- EV/sEV isolation method、corona/co-isolate、shape/orientation、batch uncertainty 必须留在 provenance。
- local-plane-wave Mie 对 20-60 nm Au/Ag 和微米级 waist 可用于 ranking；强聚焦、大粒子、near-interface phase/polarity claim 仍需 GLMT / full-wave warning。
- POD 在没有热扩散、substrate heat contribution、ROI-integrated derivative、probe/excitation split 前，只能是 paired diagnostic / frequency lane boundary。
