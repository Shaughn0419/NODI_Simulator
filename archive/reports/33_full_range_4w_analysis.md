## 1. 数据集确认

新的标准全量主库 `fine_full_range_*` 已经完成切换，当前主库口径为：

- 粒子：`gold + exosome`
- 粒径：`40–300 nm`，步进 `10 nm`
- 几何：`W = 500–2000 nm`，步进 `100 nm`；`H = 500–2000 nm`，步进 `100 nm`
- 波长：`404 / 488 / 532 / 660 nm`
- 事件数：`100 events / case`
- 总 case 数：`54 particles × 16 widths × 16 depths × 4 wavelengths = 55296`

主库状态核对：

- `meta["config_tag"] = "fine_full_range"`
- `meta["particle_profile"] = "full_range"`
- `meta["n_cases"] = 55296`
- `meta["n_events_per_case"] = 100`
- `meta["wavelengths_nm"] = [404, 488, 532, 660]`

结果健康度：

- `observation_freeze_status = default_ready_for_result_freeze`: `42336 / 55296`，约 `76.56%`
- `observation_freeze_status = caution_probe_before_result_freeze`: `12960 / 55296`，约 `23.44%`
- `engineering_gate_passed = 21308 / 55296`，约 `38.53%`

## 2. 总体结论

如果把全量主库作为一个统一结果面来观察，四个波长的角色已经明显分化：

- `404 nm` 是新的总体第一优先波长。它在全库里给出最高 `mean final_engineering_score`、最多 gate pass、最多 `recommended_default` case，也是 exosome 全材料范围内最强的主波长。
- `488 nm` 仍然是稳定的第二优先波长。它没有被 `404 nm` 取代掉工程价值，但从“主波长”降成了“强备选”。
- `660 nm` 在 gold 材料上依然非常强，尤其在浅通道和大粒径端有明显优势；但它在 exosome 上整体落后，因此不再适合作为统一主库的默认第一波长。
- `532 nm` 仍然是可用波长，但在全量主库里更多表现为第三或第四优先，主要作为大粒径补充路线。

一句话概括：

- 对 `exosome`：`404 > 488 > 532 > 660`
- 对 `gold`：`660 > 404 > 488 > 532`
- 对“统一主库默认路线”：`404 > 488 > 660 > 532`

观察：

- `532 nm` 是四波长里整体最弱的一支。

### 3.2 推荐标签分布

按 `design_recommendation_status` 聚合后，四波长的 `recommended_default` case 数分别为：

这与总体排序一致，说明 `404 nm` 不只是“偶尔冲高”，而是在推荐标签层面也成为了新的主波长。

## 6. 几何平台区

## 6.1 Exosome 的几何平台

需要注意：全量主库的几何结论和前面的 `50–150 nm focused` 扫描不同，因为现在引入了更大粒径，也包含了更宽的 `W/H` 范围。

### Exosome + 404 nm

大粒径段（`210–300 nm`）最像稳定平台的区域是：

- `W = 500–900 nm`
- `H = 800–1000 nm`

其中多个几何点在 `210–300 nm` 全段都达到 `gate_rate = 1.0`，例如：

- `500/900`
- `600/900`
- `600/1000`
- `700/800`
- `700/900`
- `700/1000`
- `900/900`

这是新的 full-range 主库里，exosome 最清晰的公共平台。

### Exosome + 488 nm

次优平台偏向：

- `W = 700–1000 nm`
- `H = 900–1000 nm`

同时在更大的几何区也出现一条偏深通道平台：

- `W = 1200–1400 nm`
- `H = 1000–1200 nm`

### Exosome + 532 nm

平台更分散，主要出现在两类区域：

- `W = 600–1000 nm`, `H = 900–1000 nm`
- `W = 1300–1500 nm`, `H = 500 / 1000–1300 nm`

说明 `532 nm` 在 exosome 上更像补充波长，而不是单一主平台波长。

### Exosome + 660 nm

最明显的平台转向浅通道大宽度：

- `W = 1200–1700 nm`
- `H = 500–600 nm`

这与 focused exosome 扫描中的结论不同。focused 扫描因为没有覆盖这些更大的 `W`，看起来 `660 nm` 主要像局部尖峰；但在 full-range 里可以看到它在“大宽度 + 浅深度 + 大粒径 exosome”上确实形成了一条窄平台。只是这条平台仍弱于 `404 nm` 主平台。

## 6.2 Gold 的几何平台

Gold 的公共平台要比 exosome 宽很多。

对大粒径 gold（`210–300 nm`）而言，四波长在下列区域几乎都很强：

- `W = 500–800 nm`
- `H = 500–800 nm`

其中：

- `404 nm` 更偏向 `500–700 / 700–1000`
- `488 nm` 更偏向 `600–800 / 500–800`
- `532 nm` 更偏向 `600–800 / 500–800`
- `660 nm` 更偏向 `600–800 / 500–800`

Gold 的几何结论比 exosome 简单得多：只要粒径足够大，较小到中等宽度、较浅到中等深度就是稳定平台。

## 7. 与旧三波长主库对比

这次四波长升级采用的是“补跑 404，再合并旧主库”的方式，因此最重要的核对点是：

1. `404 nm` 是否成功补入主库  
2. 原 `488/532/660 nm` 数据是否被污染或改变

核对结果：

- 新主库共有 `55296` 条记录，旧主库共有 `41472` 条记录
- 旧三波长子集与新四波长主库中对应子集逐条对比后：
  - `final_engineering_score` 最大绝对差值为 `4.44e-16`
  - `engineering_gate_passed` 差值为 `0`

也就是说：

- `488 / 532 / 660 nm` 的历史数据本身没有变化
- 新主库只是无损补入了 `404 nm`
- 因此四波长分析里的排序变化，是真正由 `404 nm` 带来的，而不是旧结果被重算漂移造成的

## 8. 对工程主线的含义

### 8.1 对 exosome 主线

- 默认第一波长应改为 `404 nm`
- 默认第二波长保留 `488 nm`
- `532 nm` 可作为大粒径补充路线
- `660 nm` 不建议作为 exosome 默认主波长，但可以在“大宽度 + 浅深度 + 大 exosome”这条特殊路线里保留

### 8.2 对 gold 主线

- `660 nm` 仍然值得保留为 gold 高优先波长
- `404 nm` 已经具备与 `660 nm` 并列的战略价值，尤其在统一材料路线时更重要
- 如果未来要追求“一套主系统尽量覆盖 gold + exosome”，优先级应该给 `404 nm`

### 8.3 对 dashboard 与默认结果库解释

因为标准主库已经升级为四波长，dashboard 的默认解释口径也应该同步改成：

- 默认主库波长集合：`404 / 488 / 532 / 660 nm`
- 不再把 `488 / 532 / 660 nm` 视为“标准三波长全集”
- 所有默认扫描、下拉选项、报告说明、case 数估算都应按四波长解释

## 9. 最终建议

### 统一默认建议

- 标准 full-range 主库的默认波长集合保持为：`404 / 488 / 532 / 660 nm`
- 如果需要给主线设计一个“单一默认波长”，应从 `660 nm` 切换到 `404 nm`

### 按材料的建议

- `exosome`：默认 `404 nm`，备选 `488 nm`
- `gold`：`660 nm` 与 `404 nm` 并列高优先；若强调统一主链，则优先 `404 nm`

### 按几何的建议

- exosome 主平台优先看：`W = 500–900 nm, H = 800–1000 nm`，配 `404 nm`
- gold 主平台优先看：`W = 500–800 nm, H = 500–800 nm`，其中 `404/660 nm` 最强

### 后续分析建议

- 以后凡是基于标准 `fine_full_range_*` 的图表、筛选器、摘要或 dashboard 解释，都应默认显示四波长
- 若后续要继续深挖 exosome 主线，优先对 `404 nm` 与 `488 nm` 做更高事件数 focused 重算；`532/660 nm` 更适合作为局部补充验证
