## 一句话结论

在 `1000 events/case` 的 focused sweep 下，`404 nm` 已经成为 `50–150 nm exosome` 的新主波长；`488 nm` 仍是最强备选，但不再是总体第一；`532 nm` 保留为大粒径补充波长；`660 nm` 仍主要表现为浅通道局部尖峰，不建议作为默认主设计。

### 1.1 总体排序

总体推荐顺序：

1. `404 nm`
2. `488 nm`

这个排序不是只看单点最高值，而是同时结合了：

- 平均 `final_engineering_score`

- `404 nm` 是明确第一，不仅 `max final score` 最高，而且 `gate pass` 数量达到 `122`，是 `488 nm` 的约两倍。
- `404 nm` 同时具备更高的平均检测率、更高的平均稳定检出率、更低的相位翻转上界、更大的峰值裕量，因此不是“偶然出几个高点”，而是整体工程可行性更强。
- `488 nm` 是清晰的第二名。虽然整体仍显著弱于 `404 nm`，但仍明显优于 `532 nm` 和 `660 nm`，并且在大粒径段保持了很强的竞争力。
- `532 nm` 与 `660 nm` 的差别要分开看：
  - `532 nm` 的总体均值更好，稳定性也更均衡，更适合作为第三优先波长保留。
  - `660 nm` 的 `max final score` 虽然高于 `532 nm`，但主要来自浅通道局部高点，整体均值、稳定性和平台宽度都更差。

### 4.2 总体几何趋势

- 宽度上，整体明显偏向中小宽度，最佳区大致在 `700–900 nm`。
- 深度上，不支持“越浅越好”的统一判断。
  - 对 `404 nm` 和 `488 nm`，更像是“中等深度更优”，即 `700–900 nm` 最稳。
  - 只有 `660 nm` 明显偏好浅通道 `H = 500 nm`。
- 因此全局上更合理的总结是：
  - `404/488` 主路线：中等深度更优
  - `660` 副路线：浅通道更优

### 4.4 哪些点是尖峰点

这次结果里，以下类型应视为尖峰点而不是主平台：

- `660 nm` 在 `H = 500 nm` 的若干高分点
- `532 nm` 在 `H = 900–1000 nm` 的零散高分点
- `404 nm` 的个别极高 top point，如 `150 nm, 700/700` 或 `150 nm, 800/700`

需要说明的是，`404 nm` 的高点虽然单点很高，但其周围 `700/800` 与 `800/800` 邻域依然保持正分与高 gate rate，因此 `404` 的高点并不等同于“尖峰伪优”；真正更像尖峰的是 `660` 与部分 `532` 区域。

## 5. 工程可行性结论

### 5.1 哪些区域是真的更容易通过 gate

真正“更容易通过 gate”的区域是：

- `404 nm`，`W = 700–900 nm`，`H = 700–900 nm`
- `488 nm`，`W = 700–800 nm`，`H = 800–1000 nm`

这些区域的共同特点是：

- `engineering_gate_passed` 数量高
- `engineering_basis_detection_rate` 与 `stable_detection_rate` 同时抬升
- `engineering_basis_phase_flip_fraction_wilson_ub` 较低
- `engineering_basis_mean_peak_margin_z` 大幅提高

### 5.2 哪些区域只是分数看起来高

以下区域更像“有高点，但不够稳”：

- `660 nm` 的浅通道高点
- `532 nm` 的深通道散点

原因是：

- `gate pass` 数少
- 连续粒径支持差
- 对 `W/H` 扰动敏感
- 平均表现明显低于 top point

### 5.3 关键工程量解读

#### `engineering_gate_passed`

- 是选型时最重要的硬门槛之一。
- `404 nm` 在整体和大粒径段明显领先，因此“主波长”结论首先来自这里。

#### `engineering_basis_detection_rate`

- `404` 与 `488` 在大粒径段都不差。
- 但 `404` 的 gate pass 远高于 `488`，说明它不是只靠检测率，而是整体工程条件更完整。

#### `engineering_basis_stable_detection_rate`

- `488` 在大粒径段的稳定率很接近 `404`，所以它是合格备选。
- `532` 与 `660` 稳定率虽然不至于完全失效，但不足以支撑默认主设计。

#### `engineering_basis_phase_flip_fraction_wilson_ub`

- `404` 在各个关键区间内都保持最低或接近最低，说明其相位行为更稳。
- 这也是它能把“检测率优势”真正转化成“过 gate 优势”的原因之一。

#### `engineering_basis_mean_peak_margin_z`

- 这是 `404` 相比其他波长最显著的优势项。
- 在 `120–150 nm` 段，`404` 的平均峰值裕量达到 `71.8683`，而 `488` 为 `48.8434`，`532` 为 `33.9105`，`660` 为 `23.0416`。
- 这说明 `404` 不只是可检，而是可检得更从容，工程冗余更足。

### 6.3 旧判断被强化还是被推翻

#### `488 nm` 作为主波长的旧判断

- 如果只在旧三波长体系内比较，`488 nm` 仍然是最强者，这个判断被强化了。
- 但在加入 `404 nm` 后，`488 nm` 作为“总体第一主波长”的判断被推翻了。
- 更准确的说法应该是：
  - `488 nm` 的相对地位没有变差，反而更强了
  - 但它被 `404 nm` 超越了

#### 新加入的 `404 nm`

- `404 nm` 不只是“值得进入后续主设计集合”，而是已经应当升级为第一主设计。
- 它在整体、分区间、几何平台、gate pass 数和峰值裕量上都明显领先。

#### `532 nm` 在 `130 nm` 左右的竞争力

- 这是被强化的。
- 旧结果在 `120–140 nm` 邻域中，`532 nm` 几乎没有真实过 gate 的表现。
- 新结果里：
  - `532 nm` 在 `120–150 nm` 有 `13` 个 gate pass
  - `max final score` 提升到 `1.5572`
- 但它仍然没有强到可以动摇 `488 nm` 的第二位置。

#### `660 nm` 在 `80–100 nm` 的高点

- 这类高点不完全是采样波动，因为新结果在 `80–100 nm` 确实出现了正分和少数过 gate 点。
- 但它仍然不构成宽平台，主要依赖 `H = 500 nm` 的浅通道条件。
- 因此更合理的判断是：
  - 不是纯噪声
  - 但仍然更像局部尖峰，而不是主设计路线

#### 之前怀疑的尖峰有没有变成平台

- `488 nm` 有一部分怀疑尖峰已经变成了可复现的次平台，尤其在 `W = 700–800 nm`、`H = 800–1000 nm` 一带。
- `532 nm` 的改善更多是“零散高点增加”，还没真正变成宽平台。
- `660 nm` 的改善仍主要停留在浅通道尖峰层面，不足以称为平台。

## 7. Top case 解释

本次 top 15 中，前六个点几乎全部被 `404 nm` 占据，集中在：

- 粒径：`140–150 nm`
- 宽度：`700–900 nm`
- 深度：`700–900 nm`

最高点：

- `150 nm`
- `404 nm`
- `W = 700 nm`
- `H = 700 nm`
- `final_engineering_score = 2.4491`

但本次分析不建议把这个单点直接当成默认设计点，而是把其周围的稳定平台当成真正设计区：

- `W = 700–800 nm`
- `H = 700–800 nm`

这是因为 `700/800` 与 `800/800` 这些邻域点依旧保持：

- 全邻域正分
- 全邻域过 gate
- 高稳定检出率

所以它们更适合工程落地。

## 8. 风险与不确定性

### 8.1 小粒径段仍未形成稳定平台

- `50–80 nm` 没有任何过 gate 点。
- 如果后续实验样本明显偏小粒径，目前这组参数仍不足以支持“稳定可用”。

### 8.2 `660 nm` 的浅通道高点仍需谨慎

- 它不是完全虚假，但高度依赖特定浅通道几何。
- 若实验公差较大，实际收益可能明显低于仿真中的局部高点。

### 8.3 `532 nm` 更像补充路线而非主路线

- 它已经比旧结果更强，但还没达到“默认主设计”的强度。
- 后续若继续保留，应主要面向大粒径段与较深通道区。

### 8.4 仍可进一步做的分析

若下一阶段希望更贴近真实样本分布，建议继续做：

- 粒径分布加权分析
- 只对 `404/488` 的平台区做更高事件数或更高保真局部 refine
- 针对 `660 @ H = 500 nm` 的浅通道高点做单独鲁棒性复核

## 9. 最终设计建议

- 默认主设计：
  `404 nm`，`W = 700–800 nm`，`H = 700–800 nm`
- 第二备选设计：
  `488 nm`，`W = 700–800 nm`，`H = 800–1000 nm`
- 若样本偏小粒径时的设计：
  仍优先 `404 nm`，`W = 700–900 nm`，`H = 500–700 nm`
  说明：当前只是“相对更优”，还不是稳定平台
- 若样本偏大粒径时的设计：
  `404 nm`，`W = 700–900 nm`，`H = 700–900 nm`
  如果需要第二路线，再保留 `488 nm`，`W = 700–800 nm`，`H = 800–1000 nm`
- 当前不建议优先投入的波长/几何：
  `660 nm` 作为默认主波长不建议；
  `532 nm` 不建议进入前两优先；
  多数 `W > 1000 nm` 的大宽度组合也不建议优先投入

## 10. 最终结论压缩版

- `404 nm` 已经成为这次 `50–150 nm exosome` focused sweep 的明确第一主波长。
- `488 nm` 仍然很强，但更适合当第二优先波长，而不是总体第一。
- 对真正可落地的工程选型，应优先考虑 `W = 700–900 nm`、`H = 700–900 nm` 这一公共平台区。
- `532 nm` 适合保留为大粒径补充路线。
- `660 nm` 的价值主要限于浅通道局部高点，不建议作为默认投入方向。
