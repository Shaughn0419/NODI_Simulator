## 当前状态

- dashboard 主叙事已经固定为：
  `Decision Summary -> Engineering Windows`
- 证据页和独立计算页已经从主叙事中分层：
  `Principle Guide -> Mie Explorer -> Interference Explorer -> Noise & Detection Explorer -> Design Explorer -> Case Inspector`
  以及 `Single-Case Calculator`
- 当前执行交接已经收口到 [24_高性能预计算与增量重算方案.md](../../24_高性能预计算与增量重算方案.md)

## 已完成

- 前台主阅读路径已经不再围绕 `coarse/live/fine` 混合切换组织。
- dashboard 默认结果源已经优先指向当前标准 biomimetic full-range 主库。
- 单案例页已经明确从主叙事中分离，改为独立计算工具页。
- 各页的推荐、gate、freeze、上下文提示已做过一轮共享化。
- 启动脚本已停止自动触发旧的 coarse 预计算，只提示当前标准重算命令。

## 当前原则

1. 前台主流程优先解释标准结果库，不优先解释 live rerun。
2. gold 继续作为系统验证链，exosome 才是主设计目标。
3. 推荐必须按 `gate -> recommended_default -> freeze -> final score` 读取。
4. 页面层优先复用 backend / signal_backend / common，不在页面里重复拼业务逻辑。

## 当前待办

1. 等当前标准全量库稳定后，继续收紧研究页里的静态说明和重复图表。
2. 继续减少 dashboard 页面和长版专题说明之间的重复叙述。
3. 若后续再做 UI 调整，优先改共享 header / context / verdict 语义，而不是每页各自扩写。

## 不再建议继续做的旧路线

- 不再把 `coarse` 结果库当作 dashboard 主展示入口。
- 不再走“先单独算 exosome，再与旧全量库合并”的过渡数据路线。
- 不再把 live rerun 放在前 6 页主路径里当默认操作。
