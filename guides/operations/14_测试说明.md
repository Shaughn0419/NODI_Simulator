> 文档角色：说明当前测试入口、测试分层和新增测试位置。

## 当前入口

推荐使用：

```bash
zsh tests/run_pytest_suite.sh
```

兼容旧命令：

```bash
python tests/run_tests.py --workers 8
```

`tests/run_tests.py` 现在只是轻量 wrapper，不再复制维护另一套测试实现。

## 测试分层

- `tests/test_physics_core.py`
  核心物理、信号链、读出链、评分与集成回归。
- `tests/test_dashboard_workflow.py`
  dashboard 数据源、页面默认值、工作流联动、AppTest 交互回归。
- `tests/test_ev_selected_annulus_analysis.py`
  selected-annulus EV route / ranking comparison 回归。
- `tests/test_tsuyama_gold_aligned_detection_lane.py`、`tests/test_tsuyama_selected_annulus_joint_fit.py`、`tests/test_tsuyama_2022_classification_lane.py`
  Tsuyama 对齐、joint-fit 与 classification 专项回归。
- `tests/test_event_block_randoms.py`、`tests/test_event_engine_benchmark.py`
  event-block RNG 与 benchmark helper 回归。
- `tests/dashboard_test_helpers.py`
  dashboard 测试夹具和共享断言。

默认脚本会先并行执行非 AppTest 测试，再串行执行真实 Streamlit AppTest：

```bash
python -m pytest tests -m "not app_interactions" -n 8 -q
python -m pytest tests -m "app_interactions" -q
```

## 新增测试规则

- 核心物理或信号链行为加到 `tests/test_physics_core.py`。
- dashboard 数据源、页面默认值、跨页状态和交互加到 `tests/test_dashboard_workflow.py`。
- 只有真实 Streamlit UI 交互测试才标记 `@pytest.mark.app_interactions`。
- 新测试必须避免共享固定临时路径、执行顺序依赖和跨测试可变全局状态。
