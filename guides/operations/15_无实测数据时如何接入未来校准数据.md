当前代码已经把“可以接入什么数据”和“缺什么仍不能做定量声称”分开。没有实测数据时，允许使用 `calibration/` 下的模板检查字段名和单位，但模板行都标记为 `synthetic_fixture_not_experimental`，运行时不会用它们解锁 `A_ref`、`K_sca`、absolute throughput、blank false-positive 或 detector-unit claim。

2026-04-28 复核后，manifest guard 已收紧：实测表配套 manifest 的 `calibration_kind` 必须与当前 lane 一致，例如 BFP ROI mask 必须是 `bfp_roi_mask`。错 lane manifest 会被标为 `manifest_kind_mismatch`，不会通过 calibration gate。通过 measured/calibrated contract 的 BFP ROI mask 会被投影到 Tsuyama 1D BFP q 轴并进入 `tsuyama_bfp_integrated` 的 ROI 积分；synthetic/template mask 仍只作为契约验证，不改变参考场数值。

## 现有模板

- `calibration/reference_blank_channel_template.csv`
  blank-channel `g_ref / A_ref / phi_ref_rad` 表。模板会被 `calibrated_lookup` 拒绝，避免误当实测 blank。
- `calibration/collection_operator_template.csv`
  BFP/slit/pinhole 到 `theta/phi` collection operator 的几何和 throughput 表。模板只报告 coverage，不会替代当前 surrogate operator。
- `calibration/standard_particle_template.csv`
  standard-particle `K_sca` 与 global phase offset 表。模板不会解锁 lumped `K_sca`。
- `calibration/blank_false_positive_template.csv`
  blank false-positive summary 表。模板不会替代 Gaussian/iid 或 event-background threshold。
- `calibration/raw_blank_trace_manifest_template.json`
  未来原始 blank trace bootstrap API 契约。
- `calibration/bfp_roi_mask_template.json`
  未来 BFP pixel-to-angle/ROI mask API 契约。当前模板是 inline manifest / contract fixture，不是 measured ROI mask。
- `calibration/calibration_manifest_template.json`
  实测表建议配套的 manifest 格式。

## 接入顺序

1. 复制对应模板为新的实测文件，保留列名和单位。
2. 把 `calibration_data_role` 改为 `experimental`，并配套 manifest，记录 instrument/operator/date/units/held-out validation。
3. 在 `SimulationConfig` 里只填你已具备的路径：`reference_calibration_path`、`collection_operator_calibration_path`、`standard_particle_calibration_path`、`blank_false_positive_calibration_path`、`bfp_roi_mask_path`。
4. 先跑小样本 case，检查 `*_data_role`、`*_row_id`、`*_coverage_status`、`*_synthetic_fixture_active`、`*_manifest_validation_status`、`bfp_roi_mask_gate_passed`、`detector_unit_chain_status`。
5. BFP ROI mask 通过 `bfp_roi_mask_gate_passed=True` 后，可以改变 Tsuyama BFP reference lane 的 `E_ref_complex_roi / I_ref_intensity_roi`，并导出 `tsuyama_bfp_roi_mode="calibrated_bfp_roi_mask_projected_1d"` 与 projection diagnostics；但这仍是 1D projected ROI，不等于完整 2D detector-unit overlap。没有 detector-unit chain 时仍不能输出 calibrated SNR 或 photon/voltage units。
6. 只有当 detector-unit chain、readout、count、uncertainty 和 held-out validation 全部闭环后，才允许把结果解释为定量 detector-unit 或实验计数。

## 当前边界

`schema 1.24` 会导出 row-level coverage、manifest status、manifest kind status、synthetic 防误用标记、raw blank bootstrap 边界、BFP/ROI mask 边界、detector-unit chain 边界、POD/interface 输入契约和 uncertainty propagation 边界。它们是未来实测接入的插槽，不代表当前已有实测校准。
